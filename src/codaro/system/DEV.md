# System DEV

## 역할

`src/codaro/system/`은 편집기 바깥의 로컬 시스템 기능을 API로 노출한다.

현재 두 축이 있다.
- `fileOps.py`
- `packageOps.py`

## fileOps

현재 지원 기능:
- 디렉터리 목록 조회
- 텍스트 파일 읽기
- 파일 쓰기
- 파일/디렉터리 삭제
- 이동
- 디렉터리 생성
- 존재 여부 확인

모든 경로는 `Path(...).expanduser().resolve()`를 거친다.

현재 성격은 로컬 워크스페이스 API에 가깝다.
별도 샌드박스나 project root 제한은 아직 없다.

주의점:
- 삭제와 이동이 실제 파일 시스템에 바로 반영된다
- root 제한이 없어서 추후 보안 경계 설계가 필요하다

## packageOps

현재 지원 기능:
- 설치된 패키지 목록 조회
- 패키지 설치
- 패키지 삭제

현재 구현 우선순위:
1. `uv pip`
2. 실패 시 `python -m pip`

이 폴백은 현재 코드 현실이고, 프로젝트 원칙과는 맞지 않는다.
후속 정리 대상이다.

## 프론트와의 관계

프론트는 `frontend/src/lib/editor/api.js`를 통해 이 기능을 호출한다.
장기적으로는 아래 방향으로 갈 가능성이 높다.
- 파일 트리 패널
- 패키지 패널
- AI가 파일/패키지를 조작하는 tool surface
