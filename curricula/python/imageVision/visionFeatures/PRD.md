# 이미지 비전 특징점·매칭·비디오 (visionFeatures) PRD

## 트랙 목적

이미지 한 장에서 다음 한 장(혹은 영상의 다음 프레임)으로 사고를 확장하는 트랙. 단일 이미지 처리가 끝나면 이미지 비전 응용은 거의 항상 "이 점이 다음 프레임의 어디로 갔는지", "두 사진을 어떻게 정렬할 것인지", "영상에서 무엇이 움직이는지" 같은 시간/공간 매칭으로 넘어간다. 이 트랙은 그 다리를 놓는다.

opencv 트랙(고전 CV 함수)을 마쳤거나 visionBasics를 마친 학습자를 대상으로 한다.

## 학습 후 도달 상태

- ORB로 두 사진의 특징점을 검출하고 매칭할 수 있다.
- 호모그래피로 한 사진을 다른 사진 좌표계로 옮길 수 있다.
- 스티처로 파노라마를 만들 수 있다.
- 비디오를 읽어 프레임 단위로 처리하고 저장할 수 있다.
- 배경 차분과 광학 흐름으로 움직임을 검출할 수 있다.
- CSRT/KCF 객체 트래커로 한 객체를 영상 끝까지 추적할 수 있다.

## 데이터 소스

이 트랙은 데이터셋이 풍부하지 않은 영역이라 두 가지 전략을 쓴다.

1. **이미지 매칭/스티칭**: `sklearn` china를 회전·이동시킨 합성 쌍을 만들어 사용. 외부 의존 0.
2. **비디오**: `cv2.VideoWriter` 로 색이 천천히 바뀌는 합성 mp4를 만들어 학습용으로 사용. 외부 파일 다운로드 없이 학습이 완결된다.

## 사용 라이브러리

- `opencv-python` (필수, 전 강의)
- `numpy`, `matplotlib`
- `sklearn` (load_sample_image)

SIFT는 라이선스 이슈가 있는 환경이 있으므로 ORB를 사용한다.

## 강의 작성 원칙

1. 합성 비디오는 `cv2.VideoWriter` 로 모듈 임포트 셀 다음 셀에서 만든다(첫 셀에서 만들면 다른 강의 호환성이 떨어짐).
2. cv2 함수 호출 결과 dtype·shape를 마지막 라인에 평가해 학습자가 확인 가능하게 한다.
3. 매칭 시각화는 `cv2.drawMatches`, 트래커 결과는 `cv2.rectangle` + matplotlib.
4. 비디오 출력 경로는 OS temp 디렉토리(`Path(tempfile.gettempdir()) / 'codaro_*.mp4'`) — 루트 오염 금지.
5. 셀 간 변수 재할당 금지.

## opencv 트랙과의 차이

| 항목 | opencv 트랙 | visionFeatures 트랙 |
|---|---|---|
| 데이터 | 단일 이미지 | 이미지 쌍, 영상 |
| 함수 | 필터, 에지, 컨투어 | ORB, 호모그래피, 트래커 |
| 출력 | 가공된 한 장 | 매칭 결과, 변환된 사진, 영상 |
| 사고 | 픽셀 처리 | 점·선·시간 매칭 |
