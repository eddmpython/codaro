# 건강한 중독성 설계

## 1. 정의

이 문서에서 중독성은 아이를 오래 붙잡아두는 설계가 아니다. 수상한 수학도시의 중독성은 짧은 세션이 끝난 뒤에도 다음 단서와 고쳐야 할 장소가 기억나는 힘이다.

좋은 몰입:

```text
오늘 한 에피소드를 끝냈다.
도시가 조금 고쳐졌다.
새 능력을 얻었다.
단서 하나가 남았다.
다음 장소가 궁금하다.
```

나쁜 몰입:

```text
계속하지 않으면 손해 본다.
보상이 랜덤이라 멈추기 어렵다.
순위가 떨어질까 봐 불안하다.
광고를 보면 보상을 준다.
```

금지:

- 연속 출석 보상
- 랜덤박스
- 순위표
- 광고 보상
- 제한 시간 압박
- 실패 시 재화 차감
- 끝없는 문제 피드

## 2. 몰입의 3층 구조

### 2.1 순간 몰입: 장면 속 이상함

아이의 첫 관심은 숫자나 단원이 아니라 이상한 장면이다.

예:

- 시계가 2시 40분에서 멈췄는데 버스는 3시에 출발해야 한다.
- 버스 번호가 2, 4, 6 다음에서 비어 있다.
- 손님 4명 앞에 케이크 1개가 있고 조각 크기가 다르다.

규칙:

- 문제를 내기 전에 이상함이 먼저 보여야 한다.
- 이상함은 그림으로도 이해되어야 한다.
- 단서는 최소 2개가 같은 장면 안에 있어야 한다.

### 2.2 에피소드 몰입: 작은 비밀 하나

에피소드 하나는 5-10분 안에 닫히지만, 마지막에 다음 비밀을 남긴다.

구조:

```text
사건
원인 추정
개념 능력
복구
단서
다음 예고
```

예고 문장은 1-2문장만 쓴다.

좋은 예:

```text
시계탑은 다시 움직였다.
하지만 밤이 되자 숫자 버스의 번호판에서 0이 사라졌다.
```

나쁜 예:

```text
다음 에피소드를 플레이하면 더 많은 보상을 받을 수 있어요.
```

### 2.3 장기 몰입: 내가 고친 도시

장기 동기는 점수판이 아니라 지도다.

지도에서 보여줄 것:

- 고친 장소
- 아직 이상한 장소
- 잠긴 장소
- 획득한 능력으로 열 수 있을 것 같은 장소
- 발견한 단서 수

진행률 숫자는 보조로만 쓴다. 예를 들어 `도시 복구 17%`보다 `시계탑 복구 완료, 버스 정류장 조사 가능`이 더 중요하다.

## 3. 세션 종료 설계

아이 대상 제품은 멈추기 쉬워야 한다.

에피소드 완료 후 기본 버튼:

```text
지도 보기
```

보조 버튼:

```text
다음 단서만 보기
```

`다음 에피소드 시작` 버튼은 강하게 밀지 않는다. 다음 내용을 예고하되, 지도에서 자연스럽게 들어가게 한다.

완료 화면 구성:

- 이번에 고친 장소 1개.
- 얻은 능력 1개.
- 새 단서 1개.
- 다음 이상현상 1문장.
- 지도 보기 버튼.

## 4. 재방문 동기

브라우저 저장만 쓰기 때문에 매일 접속 압박을 만들지 않는다. 대신 "돌아오면 이어서 볼 수 있음" 정도로 둔다.

재방문 요소:

| 요소 | 방식 |
| --- | --- |
| 미완성 지도 | 잠긴 장소가 실루엣으로 보임 |
| 단서장 | 단서가 연결되며 큰 비밀을 암시 |
| 능력 도감 | 얻은 능력과 아직 비어 있는 능력 표시 |
| 제로의 편지 | 다음 시즌의 감정 단서 |
| 복구 전후 | 완료 장소를 다시 눌러 변화 확인 |

하지 않는 것:

- "내일 안 오면 사라짐"
- "오늘만 보상"
- "연속 기록 유지"
- "친구보다 늦음"

## 5. 궁금증 포맷

각 에피소드는 `닫힌 답`과 `열린 질문`을 동시에 남긴다.

| 에피소드 | 닫힌 답 | 열린 질문 |
| --- | --- | --- |
| 시계탑 | 시간 차이를 알면 버스 출발을 맞출 수 있다 | 왜 0이 사라지고 있을까 |
| 버스 | 규칙을 찾으면 다음 번호를 알 수 있다 | 누가 노선도를 찢었을까 |
| 빵집 | 전체를 같은 부분으로 나누면 공평해진다 | 제로는 왜 남은 조각을 싫어할까 |

이 포맷을 지키면 에피소드가 끝나도 이야기는 앞으로 간다.

## 6. 흥미를 끄는 장치

### 6.1 오브젝트 약속

아이는 "이번 능력으로 무엇을 만질 수 있는지"를 알아야 한다.

예:

- 시간 렌즈: 시계, 시간표, 그림자.
- 패턴 렌즈: 번호판, 발자국, 기차역 표지.
- 나누기 칼: 케이크, 지도 조각, 다리 부품.

능력 도감에는 앞으로 열릴 오브젝트의 실루엣을 1개만 보여준다.

### 6.2 복구 전후 비교

장소를 완료한 뒤 다시 누르면 짧은 전후 비교를 볼 수 있다.

```text
복구 전: 시계가 멈추고 버스들이 줄 서 있음.
복구 후: 시계가 움직이고 버스가 정시에 출발함.
```

이 기능은 아이가 "내가 바꿨다"는 감각을 만든다.

### 6.3 단서 연결

단서장은 단순 수집 목록이 아니다. 단서 3개가 모이면 큰 질문 하나가 열린다.

MVP 예:

```text
단서 1: 0은 자리를 지키는 숫자야.
단서 2: 규칙을 모르면 길도 기억을 잃어.
단서 3: 나는 왜 항상 남은 조각만 받는 걸까?

큰 질문: 제로는 숫자를 없애려는 걸까, 아니면 숫자를 이해하지 못하는 걸까?
```

## 7. 에피소드 완료 포맷

모든 에피소드 완료 화면은 같은 구조를 쓴다.

```text
[장소 복구 애니메이션]
장소 복구: 시계탑 광장
새 능력: 시간 렌즈
새 단서: 0은 자리를 지키는 숫자야
다음 이상현상: 버스 번호판에서 0이 사라졌다
[지도 보기]
```

보상 텍스트보다 도시 변화 이미지가 먼저 보여야 한다.

## 8. 과몰입 방지

세션이 길어지면 쉬운 종료를 제안한다.

조건:

- 에피소드 1개 완료.
- 또는 연속 플레이 20분 이상.

문구:

```text
오늘은 시계탑을 고쳤어. 지도에서 바뀐 모습을 보고 쉬어도 좋아.
```

강제 차단은 하지 않는다. 다만 다음 에피소드를 강하게 밀지 않는다.

## 9. 품질 체크

새 기능이나 에피소드는 다음 질문을 통과해야 한다.

- 아이가 다음 버튼을 누르는 이유가 보상 때문이 아니라 궁금증 때문인가?
- 세션을 끝내기 쉬운가?
- 실패가 불안이 아니라 관찰로 이어지는가?
- 얻은 능력이 다음 장소와 연결되는가?
- 단서가 큰 비밀을 조금씩 선명하게 만드는가?
- 지도에서 내가 고친 결과가 보이는가?

## 10. 구조화 데이터 포맷

몰입 장치는 문자열 하나로 두지 않는다. 구현에서 압박형 보상으로 변질되지 않도록 구조화된 필드로 고정한다.

### 10.1 Cliffhanger

```js
const cliffhanger = {
  closedLoop: "시계탑은 다시 움직이고, 버스는 3시에 출발할 수 있게 됐다.",
  newQuestion: "그런데 밤이 되자 번호판에서 0만 사라졌다.",
  nextPlaceId: "number-bus-stop",
  nextEpisodeId: "number-bus-02",
  conceptForeshadow: "규칙",
  tone: "curious-safe",
  maxSentences: 2,
  urgency: "none",
};
```

규칙:

- 에피소드 안의 사건은 반드시 해결하고 끝낸다.
- 예고는 다음 질문 1개만 연다.
- "지금 안 보면", "놓치면", "마지막 기회" 같은 압박 문구를 쓰지 않는다.
- 다음 물건을 구체적으로 보여준다. 예: `0이 사라진 번호판`, `찢어진 지도`, `절반만 그려진 다리`.

### 10.2 지도 장소

```js
const mapPlace = {
  placeId: "clocktower",
  title: "시계탑 광장",
  state: "restored",
  beforeSymptoms: ["시계가 2시 40분에서 멈춤", "버스가 출발 시간을 모름"],
  repairAction: "시간 렌즈로 현재 시각과 출발 시각 사이를 봄",
  afterChanges: ["시계 바늘이 움직임", "버스 정류장 불이 켜짐"],
  unlockedByEpisode: "clocktower-01",
  revisitHooks: ["clocktower-pattern-revisit"],
};
```

장소 상태는 `locked`, `available`, `corrupted`, `restored`, `revisitable`까지만 둔다.

### 10.3 단서장

```js
const clue = {
  clueId: "zero-placeholder-note",
  title: "0의 쪽지",
  sourceEpisodeId: "clocktower-01",
  sourcePlaceId: "clocktower",
  text: "0은 사라지는 숫자가 아니야. 자리를 지키는 숫자야.",
  childQuestion: "0이 없으면 무엇을 읽기 어려워질까?",
  linkedConcepts: ["place-value", "pattern"],
  connectsTo: ["missing-bus-zero"],
  status: "found",
};
```

운영 규칙:

- 단서 총개수와 완성률을 전면에 세우지 않는다.
- 새 단서는 조용히 강조한다.
- 단서마다 "무엇을 알게 됐나"와 "어느 장소와 이어지나"를 보여준다.
- 아이가 다시 읽었을 때 다음 행동이 보여야 한다.

### 10.4 능력 도감

```js
const ability = {
  abilityId: "time-lens",
  name: "시간 렌즈",
  conceptLabel: "시각과 시간 차이",
  actionVerb: "두 시각 사이를 본다",
  screenEffect: "현재 시각과 목표 시각 사이가 색으로 이어짐",
  learnedInEpisode: "clocktower-01",
  canUseAt: ["clocktower", "number-bus-stop"],
  revisitUnlocks: ["bus-timetable-hidden-gap"],
  childMemoryLine: "두 시각을 함께 보면 남은 시간을 알 수 있어.",
};
```

좋은 도감은 `능력 이름`, `개념`, `쓸 수 있는 곳`, `짧은 재연`을 보여준다. 레벨, 희귀도, 별점, 연속 사용 보너스는 넣지 않는다.

### 10.5 재방문

재방문은 반복 플레이가 아니라 새 능력으로 예전 장면을 다르게 보는 것이다.

```js
const revisitHook = {
  hookId: "clocktower-pattern-revisit",
  unlockAfterAbility: "pattern-lens",
  targetPlaceId: "clocktower",
  prompt: "시계탑 종소리에도 규칙이 있을까?",
  microTaskType: "inspectScene",
  rewardType: "clueConnection",
  rewardId: "clock-chime-pattern",
  estimatedMinutes: 2,
  maxCompletions: 1,
};
```

규칙:

- 재방문 과제는 1-2분짜리 관찰 중심이다.
- 보상은 새 점수가 아니라 단서 연결, 장면 대사, 작은 지도 변화다.
- 같은 장소를 무한 반복하게 만들지 않는다.
- 재방문은 보조 동선이며 메인 에피소드 진행을 막지 않는다.

### 10.6 세션 정책

```js
const sessionPolicy = {
  targetEpisodeMinutes: 8,
  maxEpisodeMinutes: 10,
  recommendedEpisodesPerSession: 1,
  softBreakAfterMinutes: 12,
  strongBreakAfterEpisodes: 2,
  countStorage: "sessionStorage",
  breakCardPrimaryAction: "오늘은 여기까지",
  breakCardSecondaryAction: "지도만 더 보기",
};
```

에피소드 완료 화면의 기본 행동은 `오늘은 여기까지` 또는 `지도 보기`다. `바로 다음 시작`은 보조 동선으로만 둔다.
