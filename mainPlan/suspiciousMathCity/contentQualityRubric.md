# 콘텐츠 품질 루브릭

## 1. 목적

이 문서는 아이에게 보이는 점수표가 아니다. 제작자가 에피소드와 미션의 학습 품질을 판정하기 위한 내부 검수표다.

통과 기준:

- 핵심 미션은 모든 항목 평균 2점 이상이어야 한다.
- `상황 이해`, `조작`, `오개념 회복`, `접근성` 중 하나라도 1점 이하이면 출시하지 않는다.
- 재미만 있고 개념 전이가 없으면 보류한다.
- 개념은 좋지만 문제집처럼 느껴지면 보류한다.

## 2. 루브릭 표

| 기준 | 0 | 1 | 2 | 3 |
| --- | --- | --- | --- | --- |
| 상황 이해 | 단서를 못 찾음 | 단서 하나만 봄 | 필요한 단서 둘을 찾음 | 불필요한 정보도 구분 |
| 개념 선택 | 찍음 | 힌트 후 선택 | 상황에 맞게 선택 | 왜 필요한지 말할 수 있음 |
| 조작 | 무작위 조작 | 따라 함 | 맞게 조작 | 다른 숫자에도 적용 |
| 표상 연결 | 장면과 식이 분리됨 | 그림만 이해 | 그림과 식 연결 | 새 장면으로 전이 |
| 오개념 회복 | 같은 오류 반복 | 풀이 힌트 필요 | 관찰 힌트로 회복 | 스스로 오류 이유 수정 |
| 정서 반응 | 포기 | 재시도는 함 | 부담 없이 재시도 | 다음 단서로 이동 |
| 세계 반응 | 변화 없음 | 보상만 표시 | 장소 변화가 보임 | 개념 때문에 세계가 바뀐 것이 보임 |
| 접근성 | 특정 조작 없이는 불가 | 일부 대체 조작만 있음 | 키보드와 터치로 가능 | 확대, 모션 축소, 소리 없음도 통과 |

## 3. 미션 검수 기준

핵심 미션은 다음 질문을 통과해야 한다.

1. 아이가 문제 전에 무엇이 이상한지 볼 수 있는가?
2. 필요한 단서와 불필요한 단서가 구분되는가?
3. 손으로 조작하는 수학 행동이 있는가?
4. 조작이 시각 모델과 수학 문장으로 이어지는가?
5. 대표 오개념이 장면 반응으로 드러나는가?
6. 힌트가 정답을 주기보다 다시 볼 곳을 알려주는가?
7. 성공 후 세계 변화가 즉시 보이는가?
8. 같은 개념을 다른 장면에서 한 번 더 쓰는가?
9. 아이가 완료 후 배운 능력을 한 문장으로 말할 수 있는가?
10. 키보드, 터치, 큰 글자, 모션 축소에서 깨지지 않는가?

## 4. 에피소드 검수 기준

에피소드 출시 전 체크:

| 항목 | 기준 |
| --- | --- |
| 첫 30초 | 사건, 관찰, 첫 질문이 나온다 |
| 플레이 시간 | 목표 6-8분, 최대 10분 |
| 새 개념 | 1개 |
| 미션 수 | 핵심 미션 2-4개 |
| 직접 조작 | 최소 1개 |
| 오개념 반응 | 1-3개 |
| 전이 과제 | 1개 |
| 완료 화면 | 장소 복구, 능력, 단서, 기억 문장, 지도 복귀 |
| 다음 예고 | 열린 질문 1개, 최대 2문장 |
| 종료 | 자동 다음 시작 없음 |

## 5. 수동 플레이테스트 관찰표

초등 2-4학년 기준으로 관찰한다. 이름, 학년, 녹음, 원격 기록은 남기지 않는다.

| 관찰 항목 | 기록 방식 |
| --- | --- |
| 첫 30초에 이상현상을 말하는가 | 예/아니오와 짧은 메모 |
| 단서 2개를 찾는가 | 예/아니오 |
| 힌트 없이 첫 조작을 시도하는가 | 예/아니오 |
| 오답 후 다시 시도하는가 | 예/아니오 |
| 완료 후 배운 능력을 말하는가 | 아이 말 요약 |
| 그만둘 수 있는가 | 종료 버튼을 찾는지 확인 |
| 다시 오고 싶은 이유 | 이야기, 능력, 지도, 기타 중 선택 |

기록 금지:

- 아이 이름.
- 학교, 반, 연락처.
- 화면 녹화.
- 음성 녹음.
- 초 단위 행동 로그.

## 6. 출시 판정

| 판정 | 의미 | 조치 |
| --- | --- | --- |
| 통과 | 평균 2점 이상, 필수 항목 2점 이상 | 출시 가능 |
| 조건부 통과 | 평균 2점 이상이나 문장/연출 문제가 있음 | 문구와 화면만 수정 후 재검수 |
| 보류 | 필수 항목 중 1점 이하 | 에피소드 구조 수정 |
| 폐기 | 수학 개념이 세계 조작으로 연결되지 않음 | 새 brief 작성 |

출시 기준을 낮춰 에피소드 수를 늘리지 않는다. 특히 첫 시계탑 에피소드가 이 루브릭을 통과하기 전에는 다음 에피소드를 구현하지 않는다.

## 7. 출판 준비 판정

앱 출시와 출판 준비는 다르다. 출판 준비는 [publishingFormatMatrix.md](publishingFormatMatrix.md)를 기준으로 별도 판정한다.

| 기준 | 0 | 1 | 2 | 3 |
| --- | --- | --- | --- | --- |
| 이야기 전환 | 줄거리가 없음 | 앱 설명을 옮김 | 8-12쪽 이야기로 전환 | 감정과 개념이 함께 살아남 |
| 워크북 전환 | 계산 문제만 있음 | 관찰 또는 조작 하나만 있음 | 4쪽 모듈 가능 | 조작 부록까지 자연스럽게 연결 |
| 교사용 활동 | 설명서 수준 | 발문만 있음 | 15분 활동 가능 | 15/30/45분 변형 가능 |
| 흑백 품질 | 색 없으면 구분 불가 | 일부 구분 | 단서와 상태 구분 | 인쇄해도 장면이 선명 |
| 캐논 일관성 | 앱과 책 설정이 다름 | 일부 어긋남 | 같은 줄거리 유지 | 캐릭터 감정도 일관 |

출판 준비 통과:

- 모든 항목 2점 이상.
- 앱 줄거리와 책 줄거리가 어긋나지 않음.
- 워크북이 단순 반복 계산으로 변하지 않음.
- 보호자나 교사가 말 걸기 질문을 바로 사용할 수 있음.
