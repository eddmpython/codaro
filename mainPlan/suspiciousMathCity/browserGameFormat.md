# 브라우저 게임 포맷

## 1. 새 포맷

수상한 수학도시는 `라이브 수리 스테이지` 기반 정적 브라우저 퍼즐로 만든다.

```text
지도 허브
→ 고장 난 장치로 줌인
→ 수학 부품을 끼우고 작동
→ 장치 반응으로 이해
→ 복구 후 지도에 반영
```

자유 이동 게임도 아니고, 문제 카드형 학습앱도 아니다. 화면의 중심은 항상 고장 난 장치다.

## 2. 핵심 화면

### 지도 허브

지도는 전체 진행을 보여주는 허브다. 첫 진입에서는 열린 장소 1개만 강하게 보이고, 나머지는 실루엣으로 둔다.

지도에서 보여줄 것:

- 지금 고칠 수 있는 장치 1개.
- 복구된 장소.
- 다음에 궁금한 고장 장치 실루엣.
- 설정, 능력, 단서장 보조 진입.

지도에서 하지 않을 것:

- 단원 목록.
- 점수판.
- 진행률 대시보드.
- 긴 튜토리얼.

### 라이브 수리 스테이지

기본 비율:

```text
장치 스테이지 70%
조작대 30%
```

모바일에서는 다음 순서를 쓴다.

```text
장치
→ 노바 말풍선 1줄
→ 부품 트레이
→ 작동 버튼
→ 장치 반응
```

## 3. 조작 문법

| 조작 | 의미 | 대체 조작 |
| --- | --- | --- |
| 부품 집기 | 숫자, 시간, 도형, 조각을 선택 | 버튼 선택 |
| 슬롯에 끼우기 | 장치의 빈칸에 부품 배치 | 부품 선택 후 슬롯 선택 |
| 작동 | 현재 배치로 장치를 테스트 | `Enter` 또는 `Space` |
| 능력 켜기 | 숨은 구조 표시 | 토글 버튼 |
| 다시 빼기 | 배치 수정 | 삭제 버튼 또는 다시 선택 |

끌기만 요구하지 않는다. 모든 조작은 `선택 -> 슬롯 -> 작동` 방식으로도 완료되어야 한다.

## 4. 장치 반응

장치 반응이 학습 피드백이다.

| 결과 | 화면 반응 |
| --- | --- |
| 부품 부족 | 장치가 중간까지 움직이고 빈칸이 남음 |
| 부품 과다 | 장치가 목표를 지나쳤다가 되돌아옴 |
| 잘못된 부품 | 연결선이 어긋나고 슬롯이 흔들림 |
| 맞음 | 장치가 자연스럽게 작동하고 장소가 복구됨 |

문구는 장치 반응 뒤에 나온다. 먼저 보여주고, 그 다음 짧게 말한다.

## 5. 그래픽 방식

채택:

- toy-city 느낌의 상태 있는 SVG.
- HTML 조작대.
- 작은 캐릭터 표정 이미지 또는 SVG.
- 짧은 CSS transition.

핵심 오브젝트는 반드시 상태를 가진다.

```text
clockFace: stuck / focus / short / long / restored
stationLight: off / blink / on
timePart: idle / selected / placed / rejected
patternSlot: empty / wrong / fixed
cakePiece: uneven / guide / equal
```

통 bitmap은 배경 분위기나 캐릭터 표정에만 쓴다. 수학 오브젝트를 상태 없는 그림으로 고정하지 않는다.

## 6. 첫 시계버스 장치

```text
현재 시각: 2:40
출발 시각: 3:00
부품: 10분, 10분, 10분
목표: 딱 맞게 출발시키기
```

반응:

- 10분 1개: 버스가 2:50까지 가다 멈춤.
- 10분 2개: 3:00에 문이 열림.
- 10분 3개: 버스가 지나쳤다가 되돌아옴.

학습 문장:

```text
두 시각 사이 빈칸이 2개라서 20분이 남았어.
```

## 7. 저장 방식

진행 저장과 에셋 캐시는 분리한다.

| 대상 | 방식 |
| --- | --- |
| 완료 id | localStorage |
| 능력 id | localStorage |
| 단서 id | localStorage |
| 설정 | localStorage |
| 이미지와 스크립트 | 브라우저 기본 캐시 |

service worker는 첫 수리 스테이지가 검증되기 전에는 도입하지 않는다.

## 8. 모바일 기준

360px 폭에서 통과해야 한다.

- 장치가 첫 화면에서 보인다.
- 조작대가 장치를 완전히 가리지 않는다.
- 작동 버튼은 엄지로 누르기 쉽다.
- 상단바는 최소화한다.
- 아이콘 메뉴가 장면 집중을 방해하지 않는다.
- 텍스트가 버튼 안에서 잘리지 않는다.

## 9. 금지

- 문제 카드가 화면 중심이 되는 구조.
- 우측 패널에서만 답을 고르는 구조.
- 별, 점수, 레벨, 순위, 출석, 랜덤 보상.
- 완료 후 보상 목록만 보여주는 화면.
- 자동 다음 에피소드 시작.
- 긴 튜토리얼.
- 색만으로 상태를 구분하는 UI.

## 10. 구현 전 통과 조건

`시계버스 장치` graybox가 아래를 통과해야 지도와 도감 확장을 시작한다.

1. 첫 30초 안에 부품을 만질 수 있다.
2. 첫 2분 안에 작동 실패와 성공을 모두 본다.
3. 오답 이유가 장치에서 보인다.
4. 키보드만으로 부품을 끼우고 작동할 수 있다.
5. 360px 모바일에서 장치와 작동 버튼이 한 흐름으로 보인다.
