# 브라우저 게임 포맷

## 1. 제품 형태

수상한 수학도시는 GitHub Pages에서 실행되는 정적 브라우저 게임이다. 서버를 두지 않고, 진행은 접속한 브라우저 안에만 남긴다.

형태:

```text
지도 기반 허브
→ 장면 기반 에피소드
→ 오브젝트 조작 미션
→ 지도 상태 변화
```

MVP에서는 자유 이동형 게임을 만들지 않는다. 캐릭터를 방향키로 움직이는 방식은 제작 비용이 크고 학습 목표와 직접 연결되기 어렵다. 대신 지도에서 장소를 고르고, 장면 안의 오브젝트를 직접 조작하는 방식으로 간다.

## 2. 게임 방식

### 2.1 기본 플레이

```text
지도에서 열린 장소 선택
장면 관찰
노바와 짧은 대화
단서 클릭 또는 선택
수학 개념 능력 사용
오브젝트 조작
장소 복구
지도 복귀
```

### 2.2 조작 방식

MVP 조작:

| 조작 | 쓰임 |
| --- | --- |
| 선택지 클릭 | 개념 선택, 단서 선택 |
| 숫자 카드 선택 | 답 고르기 |
| 끌어놓기 | 버스 번호 순서, 조각 배치 |
| 토글 능력 | 시간 렌즈, 패턴 렌즈 켜기 |
| 오브젝트 클릭 | 시계, 번호판, 케이크 단서 찾기 |

후속 조작:

| 조작 | 쓰임 |
| --- | --- |
| 선 긋기 | 분수 자르기, 길이 재기 |
| 회전 | 각도, 톱니, 거울 |
| 막대 조절 | 그래프, 비교 |

## 3. 그래픽 방식

### 3.1 MVP 그래픽 전략

처음부터 고해상도 일러스트 전체를 만들지 않는다. 상태 변화가 중요하므로 조작 가능한 그래픽을 우선한다.

권장 조합:

```text
CSS 레이아웃
SVG 지도와 오브젝트
작은 bitmap 캐릭터 표정
CSS transition
```

이유:

- SVG는 시계 바늘, 버스 번호, 케이크 조각처럼 상태 변화가 쉽다.
- CSS는 반응형과 접근성 조정이 쉽다.
- bitmap은 캐릭터와 분위기를 살리는 데만 쓴다.
- 모든 그래픽을 이미지로 만들면 수학적 조작이 어렵다.

### 3.2 에셋 우선순위

MVP 필수 에셋:

| 에셋 | 방식 | 수량 |
| --- | --- | --- |
| 도시 지도 | SVG 또는 HTML/CSS | 1 |
| 장소 아이콘 | SVG | 6 |
| 노바 표정 | bitmap 또는 SVG | 3 |
| 제로 쪽지 | HTML/CSS | 3 |
| 능력 아이콘 | SVG | 3 |
| 시계 오브젝트 | SVG | 1 |
| 버스 번호판 | SVG/HTML | 1 |
| 케이크 조각 | SVG | 1 |
| 복구 효과 | CSS | 3 |

후속 에셋:

- 배경 음악
- 장소별 짧은 효과음
- 캐릭터 추가 표정
- 에피소드별 장면 일러스트

## 4. 저장 방식

진행 저장과 에셋 캐시는 분리한다.

| 대상 | 저장 방식 | 이유 |
| --- | --- | --- |
| 진행 상태 | `localStorage` | 작고 단순한 JSON |
| 설정 | `localStorage` | 기기별 설정 |
| 큰 그림, 소리 | 브라우저 HTTP cache | 기본 캐시로 충분 |
| 오프라인 에셋 | Cache API | 후속 PWA 단계 |
| 대용량 진행 | IndexedDB | MVP 불필요 |

진행 상태를 HTTP cache에 저장하지 않는다. cache는 브라우저가 언제든 지울 수 있고 key-value 진행 데이터에 맞지 않는다.

## 5. PWA 단계

### MVP

- manifest는 기존 `landing` manifest를 유지한다.
- `/math-city` route는 브라우저에서 바로 실행된다.
- progress는 `localStorage`.
- service worker는 등록하지 않는다.

MVP에서 service worker를 피하는 이유:

- 잘못된 캐시 전략은 오래된 화면을 보여줄 수 있다.
- 현재 목표는 오프라인보다 빠른 검증이다.
- GitHub Pages base path와 캐시 무효화 정책을 먼저 안정화해야 한다.

### PWA 1단계

조건이 맞으면 다음을 추가한다.

- `/codaro/math-city` scope에서만 동작하는 service worker.
- versioned asset cache.
- 오래된 cache 정리.
- 새 버전 감지 시 "새 이야기가 도착했어. 새로고침할까?" 문구.

주의:

- HTML은 네트워크 우선으로 둔다.
- 에셋은 캐시 우선으로 둔다.
- progress JSON은 service worker가 만지지 않는다.

## 6. GitHub Pages route 주의

`landing/vite.config.js`의 base는 `/codaro/`다. 모든 내부 링크는 `brand.appPath()` 또는 기존 app path helper를 거쳐야 한다.

주의할 경로:

```text
/codaro/math-city
/codaro/math-city/index.html
/codaro/assets/...
/codaro/math-city/...
```

정적 prerender가 필요하면 `landing/scripts/prerenderReact.js`에 `/math-city` route를 추가한다. 단순 클라이언트 route로 충분하면 처음에는 build 확인만 한다.

최종 구현 체크:

| 파일 | 필수 작업 |
| --- | --- |
| `landing/src/App.jsx` | `resolveRoute()`에 `/math-city` 추가. route만 연결하고 게임 내부 상태는 모르게 유지 |
| `landing/scripts/prerenderReact.js` | `/math-city`를 route 목록에 추가해 직접 진입 200 HTML 보장 |
| `landing/scripts/postbuild.js` | sitemap 또는 정적 URL 목록에 `/math-city` 추가 |
| `landing/vite.config.js` | base `/codaro/` 유지. 게임에서 절대경로 `/math-city/...` 직접 사용 금지 |
| `landing/static/manifest.webmanifest` | MVP에서는 기존 Codaro manifest 유지. 독립 설치 단계에서만 별도 manifest 검토 |

GitHub Pages의 `localStorage`는 origin 단위다. `/codaro` path만으로 격리되지 않으므로 storage key는 `suspiciousMathCity.progress.v1`처럼 충분히 구체적으로 둔다. 개인정보를 저장하지 않으면 이 리스크는 수용 가능하다.

## 7. 브라우저 저장 UX

저장 정책은 아이도 이해할 수 있게 말한다.

설정 문구:

```text
이 브라우저에 진행이 저장돼.
브라우저 데이터를 지우면 처음부터 다시 시작할 수 있어.
```

초기화 문구:

```text
이 브라우저의 수학도시 진행을 처음으로 돌릴까?
```

네트워크 없음 문구는 기본적으로 필요 없다. 서버 저장을 하지 않기 때문이다.

## 8. 흥미를 끄는 첫 30초

첫 30초는 설명이 아니라 사건이어야 한다.

권장 순서:

```text
0-5초: 지도에서 시계탑이 멈춰 있음
5-10초: 노바가 "시계탑이 멈췄어"라고 말함
10-20초: 버스 시간표와 시계가 함께 보임
20-30초: "무엇을 봐야 할까?" 첫 단서 질문
```

첫 30초에 하지 않는 것:

- 긴 튜토리얼
- 회원가입 안내
- 제품 설명
- 단원 목록
- 설정 요구

## 9. 첫 MVP 화면 포맷

### 지도

```text
[수상한 수학도시]
[설정]

        [시계탑: 조사 가능]
  [버스: 잠김]     [빵집: 잠김]
        [다리: 흐림]

[현재 사건 카드]
시계탑이 멈춘 날
2시 40분에서 멈춘 시계탑을 조사하자.
[조사 시작]

[능력] [단서장]
```

### 에피소드

```text
[장면: 시계탑과 버스 시간표]
노바: 시계탑이 2시 40분에서 멈췄어. 버스는 3시에 출발해야 해.

질문: 무엇을 봐야 출발까지 남은 시간을 알 수 있을까?
[현재 시각과 출발 시각]
[빵의 개수]
[동물의 키]
[가게 이름]

[힌트]
```

### 완료

```text
[시계탑 바늘이 움직임]
시계탑 광장 복구
새 능력: 시간 렌즈
새 단서: 0은 자리를 지키는 숫자야
다음 이상현상: 버스 번호판에서 0이 사라졌다
[지도 보기]
```

## 10. 구현 위험

| 위험 | 대응 |
| --- | --- |
| 카드형 문제집처럼 보임 | 장면 오브젝트를 먼저 보여주고 문제는 그 아래에 둔다 |
| 저장이 사라져 불만 | "이 브라우저에 저장" 문구와 초기화 가능한 구조 |
| service worker stale cache | MVP에서는 등록하지 않고, 후속에서 versioned cache만 사용 |
| 랜덤 문제 품질 저하 | MVP는 검수된 문제 pool 사용 |
| 모바일 겹침 | 360px 폭 스냅샷 확인 |
| 그래픽 제작 과부하 | SVG 오브젝트 우선, bitmap은 캐릭터와 분위기만 |
