# 에피소드 제작 가이드

## 1. 목적

이 문서는 새 에피소드를 만들 때 따라야 할 제작 절차다. 장기 프로젝트가 되려면 좋은 에피소드 1개를 만드는 감각이 아니라, 같은 품질의 에피소드를 계속 만들 수 있는 규격이 필요하다.

최소 단위:

```text
World Math Beat 1개
→ 에피소드 1개
→ 시즌 1개
```

## 2. 제작 파이프라인

새 에피소드는 아래 순서로 만든다.

| 순서 | 산출물 | 통과 기준 |
| --- | --- | --- |
| 1 | Episode Brief | 사건, 장소, 핵심 개념, 능력, 열린 질문이 1문장씩 있다 |
| 2 | 학습 목표 | 관찰, 조작, 개념, 표현, 전이 목표가 분리되어 있다 |
| 3 | 장면 오브젝트 | 아이가 볼 물건과 만질 물건이 id로 정의되어 있다 |
| 4 | World Math Beat | 이상현상, 수학 규칙, 단서, 조작, 모델, 반응이 있다 |
| 5 | 오개념 반응 | 대표 오개념 1-3개가 장면 반응으로 설계되어 있다 |
| 6 | 아티팩트 연결 | 능력 카드, 단서 카드, 복구 기록, 재방문 초대가 연결된다 |
| 7 | Registry 등록 | 모든 id가 중복 없이 연결된다 |
| 8 | 브라우저 확인 | 모바일, 키보드, 저장, 초기화, 모션 축소에서 깨지지 않는다 |

이 8단계를 통과하지 못하면 에피소드 수를 늘리지 않는다.

## 3. Episode Brief 템플릿

```md
## Episode: 

- episodeId:
- seasonId:
- districtId:
- 제목:
- 사건:
- 깨진 세계 규칙:
- 핵심 개념:
- 이미 배운 개념:
- 능력:
- 아이가 처음 보는 이상현상:
- 아이가 직접 만지는 오브젝트:
- 성공 후 바뀌는 세계:
- 얻는 단서:
- 닫힌 답:
- 열린 질문:
- 목표 시간:
```

작성 예:

```md
- episodeId: clocktower-01
- 사건: 시계탑이 2시 40분에서 멈췄고 버스가 출발하지 못한다.
- 핵심 개념: 시각과 시간 차이
- 능력: 시간 렌즈
- 아이가 직접 만지는 오브젝트: 10분 조각, 시계, 버스 시간표
- 닫힌 답: 두 시각 사이의 빈 칸을 보면 남은 시간을 알 수 있다.
- 열린 질문: 왜 번호판에서 0만 사라질까?
```

## 4. World Math Beat 템플릿

```js
const beat = {
  beatId: "clock-gap-01",
  type: "worldMathBeat",
  sceneObjectIds: ["clockFace", "busTimetable", "timeGapTiles"],
  brokenWorldRule: "현재 시각과 출발 시각 사이의 간격이 보이지 않는다",
  conceptId: "time-difference",
  abilityId: "time-lens",
  observationPrompt: "버스가 언제 출발해야 하는지 알려면 무엇을 봐야 할까?",
  neededClues: ["현재 시각", "출발 시각"],
  playerOperation: "10분 조각 2개를 두 시각 사이에 놓는다",
  visualModel: "2시 40분부터 3시까지 남은 칸 2개",
  mathModel: "2:40 + 20분 = 3:00",
  successWorldChange: "버스 출발등이 켜지고 시계탑 바늘이 다시 움직인다",
  misconceptionIds: ["clock-gap-full-hour"],
  memoryLine: "두 시각을 함께 보면 남은 시간을 알 수 있어.",
  transferBeatId: "bus-timetable-gap-01",
};
```

필수 규칙:

- `question/choices/answer`만 있는 미션은 금지한다.
- `playerOperation`이 없는 핵심 미션은 완료로 보지 않는다.
- `misconceptionIds`가 없는 핵심 미션은 검수 대기 상태로 둔다.
- `memoryLine`은 아이가 직접 말할 수 있는 짧은 문장이어야 한다.

## 5. 화면 진행 규격

한 에피소드는 아래 흐름을 따른다.

```text
지도에서 이상한 장소 선택
→ 장면 관찰
→ 단서 2개 찾기
→ 필요한 능력 또는 개념 선택
→ 손 조작
→ 수학 문장 확인
→ 전이 과제 1개
→ 세계 복구
→ 기억 문장
→ 단서와 다음 질문
→ 지도 복귀
```

권장 시간:

| 구간 | 목표 |
| --- | --- |
| 첫 이상현상 | 30초 이내 |
| 단서 찾기 | 1분 |
| 핵심 조작 | 2-4분 |
| 전이 과제 | 30-90초 |
| 완료 화면 | 30초 |
| 전체 | 6-8분, 최대 10분 |

## 6. 문장 작성 규칙

아이 대상 문장은 짧고 구체적이어야 한다.

| 항목 | 규칙 |
| --- | --- |
| 노바 대사 | 2문장 이하 |
| 문장 길이 | 권장 25-35자, 최대 45자 |
| 선택지 | 최대 4개 |
| 선택지 라벨 | 12자 안팎 |
| 힌트 | 볼 곳, 생각할 점, 마지막 힌트 순서 |
| 오답 문구 | 원인, 볼 곳, 다시 할 행동을 알려준다 |

나쁜 문구:

```text
틀렸어요. 다시 생각해 보세요.
```

좋은 문구:

```text
버스가 너무 일찍 도착했어.
시계에서 3시까지 남은 칸만 다시 봐.
```

## 7. 아티팩트 연결 규칙

에피소드 완료 화면은 보상 목록이 아니라 학습 기억을 남기는 화면이다.

필수 아티팩트:

| 아티팩트 | 규칙 |
| --- | --- |
| 능력 카드 | 연결 개념, 행동 동사, 쓰는 장소를 보여준다 |
| 복구 기록 | 무엇을 고쳤고 어떤 개념을 썼는지 남긴다 |
| 단서 카드 | 다음 질문과 연결되는 의미가 있어야 한다 |
| 재방문 초대 | 새 능력으로 예전 장소를 다르게 보게 한다 |
| 마무리 카드 | 오늘 배운 한 문장과 지도 복귀를 기본으로 둔다 |

금지:

- 별점, 희귀도, 연속 기록, 놓친 보상 문구.
- 다음 에피소드 자동 시작.
- 수집률을 전면에 세우는 완료 화면.

## 8. 에피소드 완료 체크리스트

새 에피소드 추가 전 아래 항목을 통과해야 한다.

1. 문제보다 이상현상이 먼저 보인다.
2. 새 개념은 1개만 중심이다.
3. 장면 단서가 최소 2개 있다.
4. 정답 선택만으로 끝나지 않고 직접 조작이 있다.
5. 장면 -> 시각 모델 -> 수학 문장 -> 다른 장면 적용 흐름이 있다.
6. 대표 오개념 1개 이상에 장면 반응과 회복 힌트가 있다.
7. 힌트는 벌점이 아니라 관찰 지원으로 작동한다.
8. 성공 후 세계 변화가 개념 이해와 직접 연결된다.
9. 완료 화면에 "오늘 배운 한 문장"이 남는다.
10. 같은 능력을 다음 에피소드나 재방문 장면에서 다시 쓴다.
11. 접근성 기준을 통과한다.
12. 저장을 지우거나 막아도 플레이가 멈추지 않는다.
13. 첫 30초 안에 "풀어야 할 문제"보다 "왜 저렇게 됐지?"를 먼저 느끼게 한다.
