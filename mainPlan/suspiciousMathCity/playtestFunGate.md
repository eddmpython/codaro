# 재미 검증 게이트

## 1. 목적

이 문서는 수상한 수학도시가 다시 카드형 문제풀이 앱으로 돌아가는 것을 막는다. 구현 품질 이전에, 아이가 실제로 만지고 싶어 하는지 확인한다.

## 2. 수동 검증 우선

초기에는 원격 분석을 넣지 않는다. 개인정보와 행동 로그를 저장하지 않는 원칙을 지킨다. 검증은 보호자 또는 제작자가 옆에서 관찰한 메모로 충분하다.

저장하지 않는 것:

- 아이 이름
- 학년
- 정답률
- 오답 횟수
- 클릭 로그
- 플레이 시간 원격 기록

## 3. 첫 2분 게이트

시계버스 graybox는 아래를 통과해야 한다.

| 기준 | 통과 |
| --- | --- |
| 첫 30초 | 아이가 설명 없이 부품이나 장치를 눌러 본다 |
| 첫 60초 | 부품을 슬롯에 끼워 본다 |
| 첫 120초 | 작동 실패와 성공을 모두 본다 |
| 말 반응 | “왜 멈췄지?”, “하나 더 넣어야 하나?” 같은 예측 말이 나온다 |
| 이탈 반응 | “문제 풀기 싫어”가 먼저 나오면 실패 |

## 4. 조작 비중 게이트

한 스테이지에서 읽기보다 조작이 길어야 한다.

실패:

```text
대화 읽기 90초
선택지 누르기 10초
결과 카드 20초
```

통과:

```text
장치 보기 10초
부품 놓기 40초
작동 실패 10초
수정 20초
작동 성공 10초
짧은 정리 10초
```

## 5. 오답 게이트

오답이 학습으로 이어지는지 본다.

| 오답 | 통과 반응 |
| --- | --- |
| 부품 부족 | “아직 빈칸이 남았네”를 볼 수 있음 |
| 부품 과다 | “지나쳤네”를 볼 수 있음 |
| 잘못된 부품 | “여기에는 다른 규칙이 필요하네”를 볼 수 있음 |

오답 후 정답을 바로 알려주면 실패다.

## 6. 확장 게이트

아래 중 하나라도 실패하면 에피소드 확장을 중단한다.

1. 핵심 스테이지가 객관식 없이 작동하지 않는다.
2. 장치 반응이 문구보다 늦거나 약하다.
3. 모바일 360px에서 장치와 조작대가 분리되어 보인다.
4. 키보드로 부품 선택과 작동을 완료할 수 없다.
5. 아이가 완료 후 “또 문제야?”라고 반응한다.

## 7. 플레이테스트 판정

내부 graybox 후, 초등 2-4학년 기준으로 최소 5명에게 수동 관찰을 권장한다. 출시 또는 시즌 확장은 아래 기준이 필요하다.

| 기준 | 목표 |
| --- | --- |
| 첫 조작 시도 | 5명 중 4명 이상 |
| 실패 후 재시도 | 5명 중 4명 이상 |
| 다음 장치 관심 | 5명 중 3명 이상 |
| 수학 말하기 | 5명 중 3명 이상이 자기 말로 설명 |

수치가 부족하면 에피소드를 늘리지 않고 첫 장치를 다시 고친다.

## 8. 문서 판정

PRD가 완성이라는 말은 아래가 닫힌 뒤에만 쓴다.

- 라이브 수리 스테이지 명세.
- 시계버스 graybox 수동 검증.
- 첫 2분 게이트 통과.
- 접근성 대체 조작 통과.
- 360px 화면 검증.

그 전에는 `리부트 설계 중`으로 표기한다.
