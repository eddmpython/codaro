# 시계버스 Episode Pack

> 이전 이름: 시계탑이 멈춘 날
> 새 역할: 라이브 수리 스테이지 기준본

## 1. 정의

이 에피소드는 시계탑 이야기를 설명하는 편이 아니다. 아이가 처음으로 `수학 부품을 끼워 넣으면 도시 장치가 움직인다`는 손맛을 경험하는 기준본이다.

| 항목 | 내용 |
| --- | --- |
| episodeId | `clockBus-01` |
| seasonId | `season-01-missing-numbers` |
| 장소 | 시계탑 광장과 버스 정류장 |
| 핵심 장치 | 멈춘 시계버스 장치 |
| 핵심 개념 | 시각과 시간 차이 |
| 선수 감각 | 10분 단위 시각, 10씩 세기 |
| 능력 | 시간 렌즈 |
| 수학 부품 | 10분 부품 |
| 닫힌 답 | 두 시각 사이 빈칸을 보면 남은 시간을 알 수 있다 |
| 열린 질문 | 왜 다음 장치에서는 0이 빠졌을까 |
| 목표 시간 | 5-7분 |

기억 문장:

```text
두 시각 사이 빈칸을 보면 남은 시간을 알 수 있어.
```

## 2. 첫 2분 루프

| 시간 | 화면 | 아이 행동 | 장치 반응 | 학습 의미 |
| --- | --- | --- | --- | --- |
| 0:00-0:20 | 지도에서 시계버스가 덜컹거리며 멈춤 | `고치기` 누름 | 장치로 줌인 | 사건 설명보다 조작 진입 |
| 0:20-0:50 | 2:40 시계와 3:00 출발판 | 10분 부품 1개를 빈칸에 놓음 | 버스가 2:50에서 멈춤 | 아직 한 칸 부족 |
| 0:50-1:20 | 빈칸 하나가 빛남 | 10분 부품 1개를 더 놓음 | 버스가 3:00에 도착 | 10분 2개가 20분 |
| 1:20-2:00 | 작동 성공 후 장치 정리 | 시간 렌즈를 켬 | 두 시각 사이가 색으로 이어짐 | 시간 차이를 시각 모델로 봄 |

첫 2분 안에 실패와 성공을 모두 보여야 한다. 아이가 한 번에 정답을 맞히더라도 `부족 예시`는 짧은 시뮬레이션으로 보여준다.

## 3. 스테이지 구성

### Stage A: 시계버스 2:40 -> 3:00

| 항목 | 내용 |
| --- | --- |
| stageId | `clockBus-gap-01` |
| 단서 | 현재 2:40, 출발 3:00 |
| 부품 | 10분 부품 3개 |
| 슬롯 | 2:40->2:50, 2:50->3:00 |
| 성공 | 슬롯 2개에 10분 부품 2개 |
| 부족 | 버스가 2:50에서 멈추고 남은 슬롯이 빛남 |
| 과다 | 버스가 3:10까지 갔다가 되돌아옴 |
| 문장 | 10분 + 10분 = 20분 |

### Stage B: 한 칸 장치 4:50 -> 5:00

| 항목 | 내용 |
| --- | --- |
| stageId | `clockBus-gap-02` |
| 목적 | 방금 배운 구조를 더 짧은 간격에 적용 |
| 부품 | 10분 부품 2개 |
| 성공 | 10분 부품 1개 |
| 부족 | 빈칸이 그대로 남음 |
| 과다 | 버스가 5:10으로 지나침 |

### Stage C: 세 칸 장치 1:30 -> 2:00

| 항목 | 내용 |
| --- | --- |
| stageId | `clockBus-gap-03` |
| 목적 | 10분 부품 3개로 30분 만들기 |
| 부품 | 10분 부품 4개 |
| 성공 | 10분 부품 3개 |
| 부족 | 1:50에서 멈춤 |
| 과다 | 2:10으로 지나침 |

### Stage D: 시간 렌즈 전이 5:20 -> 5:40

| 항목 | 내용 |
| --- | --- |
| stageId | `clockBus-transfer-01` |
| 목적 | 시간 렌즈를 다른 시간표 장치에 사용 |
| 능력 | 시간 렌즈 토글 |
| 성공 | 10분 부품 2개 |
| 완료 | 정류장 전광판이 켜짐 |

## 4. 장치 오브젝트

| 오브젝트 | 상태 |
| --- | --- |
| 시계 | `stuck`, `short`, `long`, `running`, `restored` |
| 버스 | `idle`, `moveShort`, `moveLong`, `arrived`, `repaired` |
| 출발등 | `off`, `blink`, `on` |
| 시간 렌즈 | `locked`, `installing`, `active` |
| 10분 부품 | `idle`, `selected`, `placed`, `rejected` |
| 슬롯 | `empty`, `filled`, `missing`, `overflow` |

## 5. 노바 대사

대사는 조작을 막지 않는다. 한 번에 1-2문장만 쓴다.

```text
시계는 2시 40분에서 멈췄어.
출발판은 3시를 가리켜.
```

```text
10분 부품을 빈칸에 끼워 보고 작동해 보자.
```

```text
버스가 2시 50분에서 멈췄어.
아직 한 칸이 남았네.
```

```text
이제 두 시각 사이가 보여.
이게 시간 렌즈야.
```

## 6. 제로 단서

제로는 첫 2분에 나오지 않는다. 수리 완료 뒤 다음 장치 예고로만 등장한다.

겉보기 문장:

```text
0은 없어져도 아무 일 없지 않을까?
```

복구 뒤의 질문:

```text
0이 빠지면 번호판은 무엇을 잃어버릴까?
```

다음 장치:

```text
번호판: 10 -> 1?
노바: 이번엔 0 부품이 빠졌어.
```

## 10. 제작 잠금 조건

이 Episode Pack은 아래를 통과해야만 기준본으로 인정한다.

1. 첫 30초 안에 10분 부품을 만질 수 있다.
2. 첫 2분 안에 실패와 성공을 모두 본다.
3. 정답 제출이 아니라 `작동`으로 판정한다.
4. 오답 이유가 장치 반응으로 보인다.
5. 시간 렌즈가 보상 카드가 아니라 실제 토글로 작동한다.
6. 같은 시간 차이를 다른 시간표 장치에 1회 전이한다.
7. 완료 뒤 다음 장치는 `0 부품이 빠진 번호판`으로 구체적으로 보인다.
8. 360px 모바일과 키보드 경로에서 완료 가능하다.
