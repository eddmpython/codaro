# 시계탑 Episode Pack

## 1. 정의

이 문서는 `시계탑이 멈춘 날`을 실제 제작 가능한 첫 Episode Pack으로 잠그는 기준이다. 이 한 편이 제품의 약속을 증명하지 못하면 다음 에피소드로 넘어가지 않는다.

| 항목 | 내용 |
| --- | --- |
| episodeId | `clocktower-01` |
| seasonId | `season-01-missing-numbers` |
| districtId | `clocktower-square` |
| 제목 | 시계탑이 멈춘 날 |
| 핵심 사건 | 시계탑이 2시 40분에서 멈추고, 3시에 출발해야 할 버스가 움직이지 못한다. |
| 깨진 세계 규칙 | 두 시각 사이의 시간 간격이 보이지 않는다. |
| 핵심 개념 | 현재 시각, 목표 시각, 시간 차이 |
| 함께 쓰는 개념 | 10분 단위 세기, 덧셈 |
| 능력 | 시간 렌즈 |
| 직접 조작 | 10분 조각을 시계와 시간표 사이에 놓기 |
| 닫힌 답 | 두 시각 사이의 빈칸을 보면 남은 시간을 알 수 있다. |
| 열린 질문 | 왜 번호판에서 0만 사라질까? |
| 목표 시간 | 6-8분 |

기억 문장:

```text
두 시각을 함께 보면, 남은 시간을 알 수 있어.
```

## 3. 캐릭터 대사

### 노바

노바는 정답을 말하지 않고 볼 곳을 짧게 짚는다. 한 말풍선은 1-2문장이고, 평가하는 말투를 쓰지 않는다.

```text
시계탑은 2시 40분에서 멈췄어.
버스 시간표도 같이 봐야 할 것 같아.
```

```text
두 시각 사이에 빈칸이 보여.
그 사이를 10분 조각으로 채워 보자.
```

```text
버스가 조금 일찍 움직였어.
3시까지 남은 칸만 다시 봐.
```

### 제로

제로는 이번 편에서 직접 등장하지 않고 쪽지와 흔적으로만 남는다. 무섭게 위협하지 않고, `0은 아무것도 없음`이라는 오해를 다음 사건의 수수께끼로 남긴다.

```text
0은 아무것도 없다는 뜻이잖아.
그럼 없어져도 괜찮지 않을까?
```

복구 후 다시 읽히는 의미:

```text
0은 사라지는 숫자가 아니야.
어떤 자리에서는 꼭 지켜야 하는 표시야.
```

### 탐험가

탐험가는 이름 입력을 받지 않는다. 대사보다 선택과 조작으로 존재감을 만든다.

```text
탐험가가 시간 렌즈를 켰다.
시계와 시간표 사이가 환하게 보이기 시작했다.
```

## 5. 학습 설계

### 학습 목표

| 영역 | 목표 |
| --- | --- |
| 시각 읽기 | 시계에서 정각, 30분, 10분 단위 시각을 읽는다. |
| 시간 차이 | 현재 시각과 목표 시각 사이의 남은 시간을 10분 단위로 찾는다. |
| 덧셈 | `현재 시각 + 남은 시간 = 목표 시각` 구조를 장면과 연결한다. |
| 상황 이해 | 버스 출발 문제를 해결하려면 현재 시각과 출발 시각을 함께 봐야 함을 안다. |
| 전이 | 시계탑 장면에서 배운 방식을 버스 시간표의 다른 출발 시간에도 적용한다. |

### 선수 감각

| 선수 감각 | 확인 방식 | 막힐 때 처리 |
| --- | --- | --- |
| 긴바늘과 짧은바늘 구분 | 짧은바늘은 몇 시 쪽에 있는지 보기 | 시계 읽기 미니 안내를 보여준다. |
| 정각 읽기 | 2시, 3시 같은 정각 고르기 | 정각 시계 2개 중 선택하게 한다. |
| 30분 감각 | 2시 30분, 3시 30분 읽기 | 반 바퀴 위치를 색으로 표시한다. |
| 10분 단위 감각 | 40분, 50분, 00분 위치 보기 | 시계 둘레를 10분 칸 6개로 단순화한다. |
| 10씩 더하기 | 40, 50, 60으로 세기 | 숫자보다 10분 조각을 먼저 놓게 한다. |

입장 확인은 시험처럼 보이면 안 된다. 노바가 `시계탑이 어디에서 멈췄는지 같이 보자`고 말하고, 아이가 시계를 한 번 읽는 정도로 충분하다.

### 난이도 제한

| 항목 | 허용 | 금지 |
| --- | --- | --- |
| 시각 단위 | 정각, 30분, 10분 단위 | 1분, 5분, 초 단위 |
| 시간 범위 | 같은 시간대 또는 다음 정각까지 | 여러 시간 차이, 오전/오후 전환 |
| 남은 시간 | 10분, 20분, 30분, 40분 | 50분 초과, 1시간 30분 |
| 덧셈 | 10분 단위 더하기 | 세로셈, 받아올림 중심 계산 |
| 문장 길이 | 한 문장 45자 이하 | 긴 설명문, 복합 조건 3개 이상 |
| 선택지 수 | 최대 4개 | 보기 5개 이상 |
| 새 개념 | 시간 차이 1개 중심 | 시간 단위 변환, 달력, 초 단위 동시 도입 |
| 조작 | 10분 조각 놓기 | 자유 입력만 요구 |

초기 숫자는 아래 3개로 제한한다.

```text
2:40 -> 3:00 = 20분
1:30 -> 2:00 = 30분
4:50 -> 5:00 = 10분
```

## 6. 오개념과 힌트

| 오개념 id | 잘못된 생각 | 드러나는 행동 | 장면 반응 | 회복 방향 |
| --- | --- | --- | --- | --- |
| `clock-read-hour-only` | 짧은바늘만 보고 2시라고 읽음 | 2시 40분을 2시로 고름 | 분침이 깜빡임 | 긴바늘이 가리키는 분을 다시 본다. |
| `clock-gap-full-hour` | 3시까지 무조건 60분이라고 생각함 | 60분 선택 | 색 띠가 시계 한 바퀴를 넘으려다 멈춤 | 현재 시각에서 목표 시각까지 빈 칸만 센다. |
| `clock-gap-count-target` | 목표 시각의 숫자 3만 보고 30분이라고 생각함 | 30분 선택 | 버스가 너무 일찍 출발 준비를 하다 멈춤 | 3이라는 숫자가 아니라 두 시각 사이를 본다. |
| `time-add-number-only` | 40 + 20 = 60에서 멈추고 3시로 연결하지 못함 | 60분만 답함 | 시계가 2시 60분처럼 흔들림 | 60분은 다음 정각으로 넘어간다는 장면을 본다. |
| `reading-ignore-departure` | 현재 시각만 보고 답함 | 출발 시각을 확인하지 않음 | 시간표가 흐릿하게 남음 | 현재 시각과 출발 시각 두 단서를 모두 고른다. |

힌트 사다리:

| 단계 | 목적 | 문구 |
| --- | --- | --- |
| 1 | 단서 위치를 다시 보게 함 | 시계탑과 버스 시간표를 같이 봐. |
| 2 | 두 시각 사이를 보게 함 | 지금 시각에서 출발 시각까지의 빈 칸을 찾으면 돼. |
| 3 | 10분 단위로 세게 함 | 10분 조각을 하나씩 놓아 봐. 2시 40분 다음은 2시 50분이야. |
| 4 | 수학 문장으로 연결 | 10분 조각 2개면 20분이야. 2시 40분에 20분을 더하면 3시야. |

## 7. 직접 조작 과제

### 과제 A: 멈춘 시계 읽기

목표는 현재 시각이 2시 40분임을 읽는 것이다.

보기:

```text
2시
2시 40분
3시
3시 40분
```

통과 기준:

- `2시 40분`을 고른다.
- 틀리면 긴바늘이 가리키는 40분 위치를 강조한다.

### 과제 B: 필요한 단서 고르기

질문:

```text
버스가 언제 출발할 수 있는지 알려면 무엇을 봐야 할까?
```

선택지:

```text
현재 시각
출발 시각
버스 색깔
정류장 이름
```

정답 처리:

- `현재 시각`과 `출발 시각` 두 개를 골라야 통과한다.
- 하나만 고르면 `한 가지 단서가 더 필요해`라고 안내한다.

### 과제 C: 10분 조각 놓기

장면 모델:

```text
2:40 | 2:50 | 3:00
```

아이 행동:

- 화면 아래의 `10분 조각`을 시계 둘레 빈 칸에 놓는다.
- 끌기 대신 `조각 선택 -> 빈 칸 선택 -> 놓기` 방식도 제공한다.

성공 조건:

```text
10분 조각 2개를 2:40과 3:00 사이에 놓는다.
```

성공 후 수학 문장:

```text
10분 + 10분 = 20분
2시 40분 + 20분 = 3시
```

### 과제 D: 수학 문장 완성

문장:

```text
2시 40분에 ___분을 더하면 3시가 돼.
```

보기:

```text
10
20
40
60
```

정답은 `20`이다. 이 과제는 반드시 10분 조각 조작 뒤에 등장한다.
