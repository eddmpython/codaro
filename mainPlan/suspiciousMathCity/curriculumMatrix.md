# 커리큘럼 매트릭스

> 리부트 메모: 개념 범위는 유지하되 초기 순서는 `시각과 시간 차이 -> 0과 자리값 -> 수 규칙 -> 변화량 -> 공평 배분 -> 단위분수`를 우선한다. `반쪽 빵집`은 공평 배분과 단위분수로 분리한다.

## 1. 목적

이 문서는 `수상한 수학도시`의 장기 학습 범위를 관리한다. 에피소드가 늘어나도 수학 개념, 능력, 조작, 오개념 대응, 전이 과제가 같은 품질로 유지되게 하는 기준이다.

교과 연계와 성취기준 연결은 [standardsAlignment.md](standardsAlignment.md)를 따른다. 앱, 도서, 워크북, 활동지 변환은 [crossMediaPrd.md](crossMediaPrd.md)와 [publishingFormatMatrix.md](publishingFormatMatrix.md)를 따른다.

핵심 원칙:

- 새 에피소드 하나에는 새 개념 1개만 중심으로 둔다.
- 이미 배운 개념은 조합 또는 재방문에만 쓴다.
- 모든 개념은 능력, 장면 조작, 시각 모델, 수학 문장으로 이어져야 한다.
- 단원명보다 아이가 세계에서 무엇을 보고 바꾸는지가 먼저다.

## 2. 초3 회피 학습자 기준

`으뜸이` 같은 아이는 수학 자체를 싫어한다기보다, 수학이 문제집처럼 다가올 때 피한다.

설계 기준:

| 기준 | 규칙 |
| --- | --- |
| 첫 화면 | 단원표가 아니라 이상현상이 먼저 보인다 |
| 첫 질문 | 계산이 아니라 "무엇이 이상하지?"로 시작한다 |
| 설명 길이 | 대화는 2문장 이하, 한 문장은 45자 이하 |
| 개념 수 | 한 화면의 새 개념은 1개 |
| 조작 | 계산 전 손으로 만지는 행동이 있다 |
| 실패 | 틀림 표시보다 다시 볼 단서가 먼저 나온다 |
| 기억 | 완료 후 "오늘 쓴 능력"이 한 문장으로 남는다 |

좋은 첫 반응:

```text
왜 시계가 멈췄지?
```

나쁜 첫 반응:

```text
또 문제 풀어야 해?
```

## 3. 시즌별 개념 범위

### Season 1: 사라진 숫자

목표는 초등 저학년에서 초3 초입까지 필요한 수 감각과 규칙 감각을 세계 조작으로 만든다.

| 순서 | 개념 | 선수 감각 | 에피소드 | 능력 | 조작 | 표상 사다리 | 대표 오개념 | 성공 근거 | 전이 장면 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| 1 | 시각과 시간 차이 | 시각 읽기, 10분 단위 | 시계탑이 멈춘 날 | 시간 렌즈 | 10분 조각 놓기 | 시계 -> 색 띠 -> `2:40 + 20분` | 3시까지 60분이라고 생각함 | 다른 출발 시간도 맞힘 | 버스 시간표 |
| 2 | 0과 자리값 | 십, 백의 자리 감각 | 사라진 0 | 자리 렌즈 | 10, 100, 101 번호판 복구 | 번호판 -> 자리 카드 -> `10은 1과 다르다` | 0을 지워도 비슷하다고 봄 | 10과 1의 차이를 설명 | 숫자 버스 |
| 3 | 수 규칙 | 수 순서, 같은 만큼 증가 | 숫자 버스의 실종 | 패턴 렌즈 | 번호 타일 배열 | 번호판 -> 간격 -> 수열 | 무조건 1씩 커진다고 봄 | 새 번호를 이유와 함께 고름 | 정류장 순서 |
| 4 | 공평 배분 | 같은 수로 나누기 | 반쪽 빵집 A | 공평 저울 | 빵을 사람에게 하나씩 나누기 | 접시 -> 같은 수 -> 나눗셈 말 | 남은 개수만 보고 공평하다고 봄 | 사람마다 같은 수를 받음 | 물자 나누기 |
| 5 | 등분할과 단위분수 | 전체와 부분 | 반쪽 빵집 B | 나누기 칼 | 같은 크기 자르기 | 케이크 -> 가이드선 -> `1/4` | 조각 수만 같으면 공평하다고 봄 | 크기까지 맞춰 나눔 | 지도 조각 |
| 6 | 길이 합치기 | 짧다/길다 비교 | 길이가 바뀌는 다리 | 거리 줄자 | 다리 조각 잇기 | 조각 -> 길이 막대 -> `12m + 8m` | 눈으로 길어 보이는 쪽만 선택 | 두 조각의 총 길이를 설명 | 지도 거리 |
| 7 | 빼서 남기기 | 전체와 일부 | 막힌 골목 | 제거 망치 | 장애물 수 제거 | 돌더미 -> 남은 칸 -> `15 - 6` | 남은 수와 뺀 수를 혼동 | 제거 후 남은 수를 말함 | 창고 정리 |
| 8 | 도형 기초 | 모양 비교 | 도형 공방 | 모양 도장 | 같은 모양 찾기 | 물건 -> 도형 이름 -> 분류표 | 방향이 바뀌면 다른 도형이라고 봄 | 회전해도 같은 모양임을 인식 | 간판 수리 |

목표는 곱셈, 나눗셈, 분수, 길이, 넓이를 같은 양, 반복, 전체-부분의 감각으로 연결한다.

| --- | --- | --- | --- | --- |

## 4. 에피소드 학습 목표 계약

각 에피소드 문서는 아래 항목을 반드시 가진다.

| 항목 | 작성 규칙 |
| --- | --- |
| 핵심 개념 | 새 개념 1개, 이미 배운 개념 1개 이하 |
| 선수 지식 | 아이가 이미 가져야 할 감각 |
| 관찰 목표 | 문제 전에 무엇을 봐야 하는가 |
| 조작 목표 | 손으로 무엇을 움직이는가 |
| 개념 목표 | 조작이 어떤 수학 생각으로 이어지는가 |
| 표현 목표 | 장면, 그림, 수학 문장을 어떻게 연결하는가 |
| 오개념 목표 | 일부러 드러낼 흔한 착각 1-3개 |
| 완료 근거 | 아이가 무엇을 하면 이해했다고 볼 것인가 |
| 전이 과제 | 같은 개념을 다른 장면에 한 번 더 쓰는 짧은 과제 |
| 교과 연계 | 성취기준 코드와 선수 감각 |
| 독해 함정 | 계산 오류와 구분할 상황 읽기 오류 |
| 출판 변환 | 스토리북 장면, 워크북 활동, 교사용 발문 |

## 6. 오개념 대응 레지스트리

오답은 답이 틀린 상태가 아니라 잘못된 생각이 드러난 상태다. 구현 데이터는 오답 보기보다 `wrongIdea`를 먼저 가진다.

```js
const misconception = {
  misconceptionId: "clock-gap-full-hour",
  conceptId: "time-difference",
  wrongIdea: "3시까지 60분이라고 생각함",
  triggerAnswers: ["60분", "1시간"],
  diagnosis: "현재 시각이 2시 40분이라는 단서를 놓침",
  worldResponse: "색 띠가 너무 길어져 시계를 한 바퀴 넘으려 함",
  hint1Observation: "시계탑의 분침 위치를 다시 보게 함",
  hint2Concept: "두 시각 사이의 빈 칸을 세게 함",
  hint3Procedure: "10분 칸 2개를 놓게 함",
  recoveryAction: "같은 문제를 숫자만 바꿔 재시도",
  transferCheck: "1시 30분에서 2시까지도 찾게 함",
};
```

필수 필드:

| 필드 | 의미 |
| --- | --- |
| `wrongIdea` | 아이가 가진 잘못된 생각 |
| `triggerAnswers` | 그 생각을 드러내는 선택 또는 조작 |
| `diagnosis` | 놓친 단서 또는 개념 |
| `worldResponse` | 장면에서 보이는 반응 |
| `hint1Observation` | 다시 볼 곳 |
| `hint2Concept` | 다시 생각할 개념 |
| `hint3Procedure` | 마지막 절차 힌트 |
| `recoveryAction` | 아이가 다시 할 행동 |
| `transferCheck` | 이해 전이를 확인하는 짧은 과제 |

## 7. 전이 과제 규칙

전이 과제는 같은 계산 반복이 아니다. 같은 능력을 다른 물건에 쓰는 30-90초짜리 확인이다.

좋은 전이:

```text
시계탑에서 시간 차이를 배운 뒤,
버스 시간표의 다음 출발까지 남은 시간을 찾는다.
```

나쁜 전이:

```text
20분, 30분, 40분 문제를 연속으로 더 푼다.
```

전이 과제는 에피소드 완료 조건에 포함하되, 실패해도 벌하지 않는다. 전이 과제를 못 풀면 완료 화면의 기억 문장을 더 구체적으로 보여주고 지도 복귀를 허용한다.

## 8. 출판 변환 열

커리큘럼 행은 앱만이 아니라 출판 변환 가능성도 가진다.

| 개념 | 스토리 질문 | 워크북 조작 | 교사용 발문 | 준비물 |
| --- | --- | --- | --- | --- |
| 시간 차이 | 두 시각 중 무엇을 같이 봐야 할까? | 10분 타일 색칠 | 남은 시간은 어디에서 어디까지의 간격일까? | 10분 타일 |
| 0과 자리값 | 0이 없어지면 번호가 어떻게 바뀔까? | 자리 카드 붙이기 | 10과 1은 왜 다른 번호일까? | 자리값 카드 |
| 수 규칙 | 다음 버스 번호는 왜 그렇게 될까? | 점프 선분 그리기 | 앞뒤 번호 차이가 같을까? | 번호 타일 |
| 공평 배분 | 모두 같은 수를 받았을까? | 빵 카드 나누기 | 남은 조각이 생기면 어떻게 나눌까? | 빵 카드 |
| 단위분수 | 조각 수와 크기가 모두 같을까? | 케이크 분할판 | 전체가 같을 때 한 조각은 어떻게 비교할까? | 분할판 |
