# 학습 설계

## 1. 학습 철학

수상한 수학도시의 학습은 "문제를 많이 풀면 보상을 받는다"가 아니다. 학습자는 이상한 사건을 이해하기 위해 수학 개념을 쓴다.

장기 학습 범위와 에피소드별 목표는 [curriculumMatrix.md](curriculumMatrix.md)를 따른다. 새 에피소드는 [episodeAuthoringGuide.md](episodeAuthoringGuide.md)의 제작 절차를 통과해야 하며, 출시 전 [contentQualityRubric.md](contentQualityRubric.md)의 필수 항목을 만족해야 한다.

좋은 미션은 다음 흐름을 가진다.

```text
상황 관찰
필요 개념 추론
개념 능력 획득 또는 사용
짧은 수학 미션
세계 상태 변화
단서 해금
```

아이에게 남아야 하는 기억은 "정답을 맞혔다"가 아니라 "시간 차이를 알면 시계탑을 고칠 수 있다"여야 한다.

## 2. 개념을 능력으로 바꾸는 규칙

각 개념은 도구, 행동, 세계 변화와 연결한다.

| 개념 | 능력 | 아이가 할 수 있는 일 |
| --- | --- | --- |
| 시각 읽기 | 시간 렌즈 | 멈춘 시계와 시간표의 차이를 본다 |
| 덧셈 | 합치기 장갑 | 흩어진 조각, 길이, 수량을 합친다 |
| 뺄셈 | 제거 망치 | 막힌 길에서 남은 수나 필요한 차이를 찾는다 |
| 규칙 | 패턴 렌즈 | 숫자 암호와 반복되는 길을 읽는다 |
| 나눗셈 | 공평 저울 | 사람 수에 맞게 같은 양으로 나눈다 |
| 분수 | 나누기 칼 | 전체를 반, 넷, 같은 조각으로 자른다 |
| 길이 | 거리 줄자 | 다리 조각과 지도 길이를 맞춘다 |
| 넓이 | 복구 타일 | 바닥, 농장, 방을 채운다 |
| 각도 | 회전 나침반 | 문, 거울, 톱니 방향을 맞춘다 |
| 그래프 | 분석 망원경 | 도시 사건의 수량 변화를 비교한다 |

능력 이름은 학습 단원명을 숨기기 위한 장식이 아니다. 화면 행동과 반드시 연결되어야 한다.

예:

- `패턴 렌즈`를 켜면 버스 번호판에서 2, 4, 6 다음 빈칸이 빛난다.
- `나누기 칼`을 쓰면 케이크 위에 같은 크기 가이드 선이 나타난다.
- `시간 렌즈`를 쓰면 현재 시각과 출발 시각 사이가 색으로 표시된다.

## 2.1 개념 능력 계약

개념 능력은 도감 보상이 아니라 화면 조작 문법이다. 아이는 능력을 켜서 장면의 숨은 수학 구조를 보고, 직접 움직이고, 세계 반응으로 맞는지 확인한다.

새 능력은 다음 포맷을 반드시 채운다.

```md
- 능력 이름:
- 연결 개념:
- 깨진 세계 규칙:
- 처음 보이는 이상현상:
- 능력을 켰을 때 드러나는 것:
- 아이가 직접 조작하는 것:
- 연결되는 시각 모델:
- 연결되는 수학 문장:
- 성공하면 회복되는 세계 상태:
- 다른 에피소드에서 다시 쓰이는 방식:
```

MVP 능력 계약:

| 능력 | 깨진 세계 규칙 | 드러나는 것 | 조작 | 수학 문장 |
| --- | --- | --- | --- | --- |
| 시간 렌즈 | 두 시각 사이의 간격이 보이지 않음 | 현재 시각과 목표 시각 사이의 빈 칸 | 10분 조각을 사이에 놓기 | `2:40 + 20분 = 3:00` |
| 패턴 렌즈 | 같은 만큼 커지는 규칙이 보이지 않음 | 번호 사이의 간격 | 다음 번호 타일 놓기 | `2, 4, 6, 8, 10` |
| 나누기 칼 | 전체가 같은 부분으로 나뉘지 않음 | 같은 크기 가이드 선 | 케이크를 4조각으로 나누기 | `1개를 4명이 나누면 한 명은 1/4` |

능력은 획득한 뒤 끝나지 않는다. 다음 에피소드나 재방문 장면에서 다시 쓰여야 한다.

## 2.2 표상 사다리

모든 핵심 미션은 다음 순서로 설계한다.

```text
장면 오브젝트
→ 시각 모델
→ 수학 문장
→ 다른 장면 적용
```

수식은 먼저 나오지 않는다. 아이가 조작한 장면을 정리하는 언어로 나온다.

예:

```text
케이크를 4조각으로 자른 뒤에야 1/4이라는 말이 나온다.
아이는 분수를 외우는 것이 아니라, 같은 크기로 나뉜 세계를 이름 붙인다.
```

시계탑 예:

```text
시계 바늘과 시간표를 본다.
→ 2시 40분부터 3시까지 빈 칸이 색으로 보인다.
→ 20분이 남았다고 말한다.
→ 다른 버스 출발 시간에도 같은 방식으로 적용한다.
```

## 3. 미션 유형

### 3.1 단서 선택

문제 풀이 전 단계다. 아이가 상황에서 필요한 정보를 고르게 한다.

예:

```text
버스가 3시에 출발해야 해.
지금 시계는 2시 40분이야.

무엇을 알아야 버스가 언제 출발하는지 알 수 있을까?
```

선택지:

- 현재 시각과 출발 시각
- 빵의 개수
- 동물의 키
- 가게 이름

학습 목표: 문제를 읽고 필요한 정보와 불필요한 정보를 구분한다.

### 3.2 개념 선택

상황에 필요한 개념을 고른다.

예:

```text
손님 4명이 케이크 1개를 똑같이 나누고 싶어 한다.
어떤 힘이 필요할까?
```

선택지:

- 합치기
- 나누기
- 회전하기
- 그래프 보기

학습 목표: 개념 이름이 아니라 쓰임을 이해한다.

### 3.3 짧은 계산

계산은 길지 않게 유지한다. 계산 결과가 바로 세계 변화로 이어져야 한다.

예:

```text
지금은 2시 40분.
버스는 3시에 출발.
몇 분 뒤에 출발할까?
```

학습 목표: 시간 차이를 실제 상황에 적용한다.

### 3.4 배열과 조작

초등학생에게 강한 상호작용은 숫자 입력보다 끌어놓기다.

예:

- 버스 정류장 번호 `2, 4, 6, 8, ?`를 순서대로 놓기.
- 빵 조각 4개를 손님 4명에게 하나씩 나누기.
- 다리 조각 12m와 8m를 합쳐 총 길이 칸에 놓기.

학습 목표: 머릿속 계산을 화면 조작과 연결한다.

## 4. 힌트 설계

힌트는 정답 노출이 아니라 관찰을 돕는다.

힌트 단계:

1. 관찰 힌트: 장면에서 무엇을 봐야 하는지 알려준다.
2. 개념 힌트: 어떤 능력이 필요한지 알려준다.
3. 풀이 힌트: 식이나 단계 일부를 보여준다.

예:

```text
힌트 1: 버스 시간표와 시계탑을 같이 봐.
힌트 2: 두 시각 사이의 차이를 찾으면 돼.
힌트 3: 2시 40분에서 3시까지는 20분이야.
```

힌트 사용은 벌점으로 처리하지 않는다.

## 5. 난이도 규칙

MVP 숫자 범위:

| 개념 | 범위 |
| --- | --- |
| 시각 | 정각, 30분, 10분 단위 |
| 덧셈 | 20 이하, 받아올림 없음 |
| 규칙 | 1, 2, 5씩 증가 또는 감소 |
| 나눗셈 | 2명, 4명에게 똑같이 나누기 |
| 분수 | 1/2, 1/4, 전체와 부분 |

에피소드 하나에 새 규칙을 여러 개 넣지 않는다. 새 개념 1개와 이미 배운 개념 1개 조합이 최대다.

## 6. 에피소드별 학습 목표

### Episode 1: 시계탑이 멈춘 날

목표:

- 시계에서 현재 시각을 읽는다.
- 목표 시각까지 남은 시간을 찾는다.
- 시간 차이를 덧셈 또는 뺄셈으로 표현한다.

미션 순서:

1. 시계탑과 버스 시간표 관찰.
2. 필요한 정보 선택.
3. 2시 40분에서 3시까지 남은 시간 찾기.
4. 다른 버스 2대의 출발 시간 맞추기.
5. 시간 렌즈 획득.

### Episode 2: 숫자 버스의 실종

목표:

- 수열의 증가 규칙을 찾는다.
- 빈칸에 들어갈 수를 예측한다.
- 규칙이 도시 이동 순서와 연결된다는 것을 이해한다.

미션 순서:

1. 사라진 버스 번호판 관찰.
2. 번호들이 어떤 규칙으로 커지는지 선택.
3. 빈 버스 번호 채우기.
4. 정류장 순서 배열.
5. 패턴 렌즈 획득.

### Episode 3: 반쪽 빵집

목표:

- 전체와 부분을 구분한다.
- 같은 크기로 나눈다는 뜻을 이해한다.
- 1/2, 1/4를 장면에서 인식한다.

미션 순서:

1. 손님 수와 케이크 수 관찰.
2. 필요한 개념 선택.
3. 케이크를 2등분, 4등분으로 나누기.
4. 손님에게 같은 양 배분.
5. 나누기 칼 획득.

## 7. 오답 처리

오답은 실패가 아니라 세계가 아직 불안정하다는 피드백으로 보여준다.

예:

- 잘못된 버스 번호를 넣으면 버스가 한 칸 갔다가 돌아온다.
- 케이크를 불공평하게 나누면 손님 표정이 어리둥절해지고, 조각 크기 차이가 색으로 표시된다.
- 시간 차이를 틀리면 시계 바늘이 목표 시각을 지나쳤다가 되돌아온다.

문구는 짧고 구체적으로 쓴다.

나쁜 문구:

```text
틀렸어요.
```

좋은 문구:

```text
버스가 너무 일찍 도착했어. 시계탑에서 3시까지 남은 시간을 다시 봐.
```

## 8. 콘텐츠 품질 체크

에피소드를 추가할 때 다음을 만족해야 한다.

- 문제 전 장면 단서가 있다.
- 필요한 개념을 고르는 질문이 있다.
- 계산 미션이 2-4개를 넘지 않는다.
- 성공 후 세계 변화가 있다.
- 단서 또는 다음 예고가 있다.
- 같은 에피소드 안에서 새 개념이 2개를 넘지 않는다.
- 오답 피드백이 정답만 알려주지 않는다.
- 저장을 지우고도 다시 플레이 가능하다.

추가 출시 기준:

- 핵심 미션은 루브릭 평균 2점 이상이어야 한다.
- `상황 이해`, `조작`, `오개념 회복`, `접근성` 중 하나라도 1점 이하이면 출시하지 않는다.
- 새 에피소드 하나에는 새 개념 1개만 중심으로 둔다.
- 같은 능력이 다음 에피소드나 재방문 장면에서 다시 쓰여야 한다.
- 전이 과제가 없는 에피소드는 학습 완료로 보지 않는다.
