# 접근성과 초등 사용성 기준

## 1. 목적

이 게임은 초등학생이 혼자 들어와도 쓸 수 있어야 한다. 접근성은 별도 배려 기능이 아니라 MVP 완료 조건이다.

기준:

- 키보드만으로 완주 가능.
- 터치 조작이 아이 손가락 크기에 맞음.
- 끌기 과제에는 대체 조작이 있음.
- 색, 소리, 빠른 모션에 의존하지 않음.
- 글자가 커져도 화면이 깨지지 않음.
- 실패 후 무엇을 다시 봐야 하는지 알 수 있음.

참고 기준은 WCAG 2.2와 웹 저장소 표준 문서를 따른다. 구현은 이 문서를 실제 검수표로 사용한다.

## 2. MVP 접근성 완료 조건

| 영역 | 완료 조건 |
| --- | --- |
| 키보드 | 지도, 선택지, 능력, 힌트, 단서장, 설정, 초기화까지 `Tab`, `Enter`, `Space`, 화살표로 조작 가능 |
| 포커스 | 포커스 순서가 읽기 순서와 같고, 포커스 테두리가 명확함 |
| 터치 | 주요 터치 목표는 `48x48 CSS px` 이상, 보조 버튼도 `44x44` 이상 |
| 간격 | 인접 버튼 간격은 최소 `8px` |
| 끌기 대체 | 모든 끌기 과제는 `타일 선택 -> 목표 칸 선택 -> 놓기`로도 해결 가능 |
| 색 대비 | 본문 4.5:1 이상, 큰 글자 3:1 이상, 비텍스트 상태 3:1 이상 |
| 색 의존 | 잠김, 조사 가능, 복구됨, 다시 볼 수 있음은 색+아이콘+텍스트로 표시 |
| 모션 | `prefers-reduced-motion`과 게임 설정 `reducedMotion`을 모두 따름 |
| 오디오 | 소리 없이도 성공, 오답, 단서, 완료 정보를 모두 이해 가능 |
| 확대 | 브라우저 200% 확대와 `largeText` 모드에서 가로 스크롤 없이 플레이 가능 |
| 모바일 | 360px 폭에서 버튼, 대화, 하단 시트가 겹치지 않음 |

## 3. 초등 단독 사용 기준

| 항목 | 규칙 |
| --- | --- |
| 화면 목표 | 화면마다 "지금 할 일" 한 줄을 고정한다 |
| 대화 | 대사는 2문장 이하 |
| 문장 길이 | 권장 25-35자, 최대 45자 |
| 선택지 | 최대 4개 |
| 새 개념 | 한 화면 1개 |
| 튜토리얼 | 첫 30초 안에 사건, 관찰, 첫 질문이 나온다 |
| 도움 | `힌트`, `다시 보기`, `처음부터`가 항상 보인다 |
| 종료 | `지도 보기` 또는 `오늘은 여기까지`가 명확하다 |

장면, 대화, 조작판, 힌트를 한 번에 모두 펼치지 않는다. 아이가 지금 해야 할 행동 하나만 먼저 보이게 한다.

## 4. 컴포넌트 기준

| 컴포넌트 | 필수 기준 |
| --- | --- |
| `CityMapPlace` | 버튼 요소로 구현. 상태는 `잠김`, `조사 가능`, `복구됨`, `다시 볼 수 있음` 텍스트를 포함 |
| `EpisodeScene` | 핵심 단서 오브젝트는 키보드 포커스 가능. 포커스 시 테두리와 짧은 설명 표시 |
| `ChoiceTask` | 선택지는 버튼. 선택 후 바로 이동하지 않고 결과 피드백을 보여준다 |
| `DragArrangeTask` | 끌기, 탭 선택, 키보드 이동 3가지 조작을 제공 |
| `SplitWholeTask` | 선 긋기만 요구하지 않고 `2등분`, `4등분` 단계 버튼과 미리보기를 제공 |
| `AbilityToggle` | 켜짐/꺼짐 상태를 텍스트로 함께 표시 |
| `HintPanel` | `볼 곳 -> 생각할 점 -> 마지막 힌트` 3단계 |
| `FeedbackMessage` | "틀림" 단독 금지. 원인, 볼 곳, 다시 할 행동을 제공 |
| `BottomSheet` | 손잡이, 닫기 버튼, 제목, 포커스 복귀 규칙 필수 |
| `SettingsPanel` | 글자 크게, 모션 줄이기, 소리, 진행 초기화. 초기화는 확인 화면 필수 |

## 5. 하단 조작판 기준

모바일에서는 하단 조작판이 학습의 중심이다.

상태:

| 상태 | 역할 |
| --- | --- |
| `peek` | 현재 목표와 핵심 버튼만 보임 |
| `half` | 선택지 또는 타일 조작을 보여줌 |
| `full` | 단서장, 도감, 설정처럼 읽을 내용이 많을 때 사용 |

규칙:

- 하단 조작판은 핵심 장면과 현재 목표를 완전히 가리지 않는다.
- 화면 하단 안전 여백을 적용한다.
- 열린 조작판 안에서 포커스가 길을 잃지 않아야 한다.
- 닫기 버튼 없이 배경 클릭만으로 닫히게 하지 않는다.
- 조작판이 열렸을 때 이전 포커스 위치로 돌아갈 수 있어야 한다.

## 6. 모션과 시각 반응

모션은 의미가 있어야 하고 짧아야 한다.

| 반응 | 일반 모드 | 모션 축소 모드 |
| --- | --- | --- |
| 단서 강조 | 300ms pulse | 테두리와 라벨 |
| 오답 | 180ms nudge | 색+문구 |
| 정답 | 240ms snap | 즉시 상태 전환 |
| 장소 복구 | 1-2초 애니메이션 | 복구 전후 1프레임 전환 |
| 완료 연출 | 건너뛰기 가능 | 자동 건너뛰기 또는 정지 상태 |

반복 애니메이션은 자동으로 멈춘다. 빠른 깜박임, 긴 흔들림, 계속 움직이는 배경은 쓰지 않는다.

## 7. 저장 실패와 회복

저장소는 영구 보장이 아니다. 아이가 저장 문제 때문에 막히면 안 된다.

화면 문구:

```text
이 브라우저에는 저장하지 못했어.
그래도 지금 이야기는 계속할 수 있어.
```

처리:

- 저장 실패 시 게임 진행은 메모리 상태로 계속된다.
- 깨진 저장값은 기본 상태로 복구한다.
- 초기화는 확인 화면을 거친다.
- 이름, 학년, 자유입력, 기기 식별자는 저장하지 않는다.

## 8. 접근성 테스트 체크리스트

구현 후 아래 항목을 통과해야 한다.

1. 키보드만으로 지도 진입, 에피소드 시작, 문제 풀이, 힌트 열기, 완료, 지도 복귀가 된다.
2. 모든 끌기 과제는 끌지 않고도 같은 결과를 낼 수 있다.
3. 360px, 390px, 414px 폭에서 버튼 텍스트와 하단 시트가 겹치지 않는다.
4. 브라우저 200% 확대와 `largeText` 모드에서 가로 스크롤 없이 완료 가능하다.
5. 본문, 버튼, 아이콘, 지도 상태, 포커스 링의 대비 기준을 통과한다.
6. 회색조로 보아도 잠김, 가능, 복구, 오답, 정답 상태가 구분된다.
7. 모션 줄이기 모드에서 모든 과제가 동일하게 풀린다.
8. 소리를 끄거나 사용할 수 없어도 단서, 성공, 오답, 완료 정보가 모두 전달된다.
9. 저장 실패, 깨진 저장값, 진행 초기화 후에도 앱이 멈추지 않는다.
10. 초등 2-4학년 독해 기준으로 한 화면 목표를 5초 안에 말할 수 있다.
