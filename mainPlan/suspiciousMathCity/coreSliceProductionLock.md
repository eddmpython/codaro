# Core Slice 제작 잠금

## 1. 목적

이 문서는 `시계탑이 멈춘 날` Product Core Slice의 화면, 조작, 그래픽, 에셋, 접근성 기준을 잠근다. 완료 기준은 문제 풀이 화면이 보이는 것이 아니라, 시간 차이를 알아야 세계가 복구된다는 감각이 화면과 조작으로 증명되는 것이다.

경험 순서:

```text
지도에서 멈춘 시계탑 발견
→ 시계탑 장면 진입
→ 현재 시각과 버스 출발 시각 관찰
→ 시간 차이를 조작으로 확인
→ 시계탑과 정류장 복구
→ 시간 렌즈와 0 단서 획득
→ 다음 장소가 궁금해짐
```

## 2. 화면 상태표

| 화면 | 상태 | 보여야 하는 것 | 가능한 행동 | 완료 조건 |
| --- | --- | --- | --- | --- |
| 지도 | `fresh` | 도시, 멈춘 시계탑, 잠긴 장소 | 시계탑 선택, 설정 열기 | 시계탑이 첫 진입 대상임을 5초 안에 알 수 있음 |
| 지도 | `clocktower-corrupted` | 시계탑 바늘 멈춤, 정류장 불 꺼짐 | 시계탑 에피소드 시작 | 장소 상태가 색 없이도 구분됨 |
| 지도 | `clocktower-restored` | 시계탑 색 복구, 정류장 불 켜짐 | 다시 보기, 단서장 열기, 다음 장소 확인 | 복구 전후가 한눈에 다름 |
| 시계탑 장면 | `intro` | 2시 40분 시계, 3시 버스 시간표, 노바 대화 | 단서 오브젝트 탐색 | 핵심 단서 2개가 같은 화면에 보임 |
| 시계탑 장면 | `inspect` | 포커스된 시계 또는 시간표 설명 | 시계 보기, 시간표 보기, 힌트 열기 | 현재 시각과 목표 시각을 말할 수 있음 |
| 태스크 카드 | `story-choice` | 무엇을 봐야 하는지 묻는 질문 | 선택지 4개 중 선택 | 상황 정보 선택이 계산보다 먼저 나옴 |
| 태스크 카드 | `time-gap` | 10분 조각, 시계 눈금, 버스 출발 시각 | 끌기 또는 선택식 배치 | 2시 40분에서 3시까지 20분임을 조작으로 확인 |
| 태스크 카드 | `wrong` | 바늘이 너무 짧거나 길게 이동, 원인 문구 | 다시 시도, 힌트 보기 | 꾸짖지 않고 다시 볼 단서를 알려줌 |
| 보상 | `ability-reveal` | 시간 렌즈, 개념명, 쓸 수 있는 곳 | 도감 보기, 지도 보기 | 능력이 장식이 아니라 다음 조작 언어로 이해됨 |
| 단서장 | `clue-reveal` | 0이 사라지고 있다는 쪽지 | 단서 저장, 다음 예고 보기 | 다음 에피소드의 궁금증을 남김 |
| 설정 | `settings` | 소리, 큰 글자, 모션 줄이기, 진행 초기화 | 토글, 초기화 확인 | 개인정보 없이 브라우저 진행만 다룸 |

## 3. 조작 계약

| 조작 대상 | 기본 조작 | 대체 조작 | 키보드 조작 |
| --- | --- | --- | --- |
| 지도 장소 | 터치/클릭 | 장소 목록 버튼 | `Tab` 이동, `Enter` 시작 |
| 시계 오브젝트 | 터치/클릭으로 단서 보기 | 단서 목록에서 선택 | `Tab` 포커스, `Enter` 설명 |
| 시간표 | 터치/클릭 | 단서 목록에서 선택 | `Tab`, `Enter` |
| 10분 조각 | 끌어서 눈금에 놓기 | `조각 선택 -> 칸 선택 -> 놓기` | 화살표로 칸 이동, `Space` 배치 |
| 선택지 | 버튼 선택 | 동일 | 화살표 또는 `Tab`, `Enter` |
| 힌트 | 버튼 | 항상 표시 | `Tab`, `Enter` |
| 하단 시트 | 손잡이 또는 버튼 | 닫기 버튼 | `Esc` 닫기, 닫은 뒤 이전 포커스 복귀 |

끌기만 요구하지 않는다. 모든 조작형 과제는 선택 기반으로도 완료되어야 한다.

## 4. 그래픽 방식

| 영역 | 방식 | 이유 |
| --- | --- | --- |
| 도시 지도 | 반응형 SVG + HTML 버튼 레이어 | 장소 상태, 포커스, 텍스트 설명을 같이 관리 |
| 시계탑 장면 | inline SVG 오브젝트 | 바늘, 시간 띠, 정류장 불 상태를 직접 변경 |
| 태스크 카드 | HTML 버튼과 SVG 조작판 혼합 | 글자 확대와 키보드 접근성 확보 |
| 시간 렌즈 효과 | SVG 색 띠 + 눈금 라벨 | 시간 차이를 시각 모델로 보여줌 |
| 노바 | 작은 SVG 또는 1x/2x 표정 이미지 | 감정 보조, 퍼즐 방해 금지 |
| 제로 쪽지 | HTML/CSS 또는 SVG | 단서장과 인쇄 확장에 유리 |
| 보상 연출 | 짧은 상태 전환 | 폭죽보다 세계 변화 중심 |

지도 전체를 한 장의 그림으로 고정하지 않는다. 핵심 수학 오브젝트는 반드시 상태와 포커스를 가진다.

## 5. 에셋 인벤토리

| assetId | 형식 | 상태 | 쓰는 화면 | 필수 |
| --- | --- | --- | --- | --- |
| `mapCityCore` | SVG | 기본, 시계탑 이상, 시계탑 복구 | 지도 | 예 |
| `placeClocktower` | SVG group | 가능, 이상, 복구, 다시 보기 | 지도 | 예 |
| `sceneClocktower` | SVG | 이상, 복구 | 에피소드 | 예 |
| `clockFace` | SVG object | 멈춤, 포커스, 오답, 정답, 복구 | 에피소드 | 예 |
| `busTimetable` | SVG/HTML | 흐림, 포커스, 확인됨 | 에피소드 | 예 |
| `minuteTiles10` | SVG/HTML | 대기, 선택, 배치, 거절, 확정 | 태스크 | 예 |
| `timeLensIcon` | SVG | 잠김, 획득 | 보상, 도감 | 예 |
| `novaExpressionSet` | SVG 또는 bitmap | 궁금, 놀람, 기쁨 | 대화 | 예 |
| `zeroMissingNote` | HTML/SVG | 접힘, 펼침 | 단서장 | 예 |
| `settingsIcons` | SVG | 기본 | 설정 | 예 |

후속 보강 자산은 배경 질감, 종소리, 복구 빛 효과다. 단, 소리와 장식 효과는 학습 흐름을 대체하지 않는다.

## 6. 레이아웃

| 기준 | 모바일 | 데스크톱 |
| --- | --- | --- |
| 기본 구조 | 세로 단일 흐름 | 지도/장면 + 우측 패널 |
| 지도 | 화면 폭 전체, 장소 3-6개 구분 | 넓은 지도와 단서장 동시 표시 |
| 에피소드 | 장면 -> 대화 -> 하단 조작판 | 장면 좌측, 조작판 우측 가능 |
| 하단 시트 | `peek`, `half`, `full` | 패널 또는 drawer |
| 도감/단서장 | 하단 탭 | 우측 패널 |
| 설정 | 전체 시트 | 모달 또는 패널 |

360px 폭에서 버튼 텍스트가 잘리지 않아야 한다. 큰 글자 모드에서도 가로 스크롤 없이 완료 가능해야 한다.

## 7. 모션 계약

| 상황 | 기본 모션 | 모션 줄이기 |
| --- | --- | --- |
| 단서 강조 | 300ms 테두리 강조 | 정적 테두리와 라벨 |
| 오답 | 180ms 짧은 흔들림 | 문구와 상태 아이콘 |
| 정답 | 240ms 위치 고정 | 즉시 고정 |
| 시계 복구 | 1.2초 바늘 이동 | 복구 상태로 즉시 전환 |
| 지도 복귀 | 장소가 밝아짐 | 복구 배지 표시 |

반복 애니메이션은 자동으로 멈춘다. 빠른 깜박임, 긴 흔들림, 계속 움직이는 배경은 쓰지 않는다.

## 8. 접근성 인수조건

| 영역 | 인수조건 |
| --- | --- |
| 키보드 | 지도 진입부터 에피소드 완료, 보상 확인, 지도 복귀까지 마우스 없이 가능 |
| 터치 | 주요 목표 `48x48 CSS px` 이상, 보조 버튼 `44x44` 이상 |
| 끌기 대체 | 모든 끌기 과제는 선택 기반 배치로도 완료 |
| 포커스 | 포커스 순서가 화면 읽기 순서와 같고, 현재 위치가 분명함 |
| 색 의존 | 잠김, 가능, 이상, 복구, 정답, 오답은 색 외에 아이콘 또는 텍스트 포함 |
| 글자 크게 | 큰 글자와 200% 확대에서 가로 스크롤 없이 완료 |
| 모션 줄이기 | 모든 학습 흐름과 완료 판정이 동일하게 가능 |
| 소리 없음 | 종소리 없이도 성공과 단서 획득을 이해 |
| 저장 실패 | 저장이 막혀도 현재 플레이는 계속되고 짧은 안내가 표시 |
| 초기화 | 진행 초기화는 확인 화면을 거치며 개인정보 관련 문구 없음 |

## 9. 제작 잠금 판정

시계탑 Core Slice는 아래 조건을 모두 만족해야 잠금 처리한다.

1. 첫 30초 안에 이상현상, 핵심 단서 2개, 첫 질문이 나온다.
2. 시간 차이 개념이 선택지가 아니라 장면 조작으로 드러난다.
3. 오답 피드백이 왜 다시 봐야 하는지 보여준다.
4. 완료 후 시계탑과 지도 상태가 눈에 띄게 바뀐다.
5. 시간 렌즈가 도감에 남고, 다음 장소에서 다시 쓸 수 있을 것처럼 보인다.
6. 단서장은 다음 이야기의 질문을 남기되 자동으로 다음 에피소드를 시작하지 않는다.
7. 모바일 360px, 데스크톱 1280px, 큰 글자, 모션 줄이기, 키보드 경로가 모두 통과한다.
8. 지도, 장면, 태스크, 보상, 단서장, 설정이 모두 같은 시각 언어를 쓴다.
9. 학습 없이 장식만 있는 화면이 없고, 장면 없이 문제만 있는 화면도 없다.
