# 구현 인수조건

## 1. 현재 판정

기존 인수조건은 `지도 + 에피소드 패널 + task renderer`를 전제로 했다. 리부트 후 구현 기준은 `라이브 수리 스테이지`다.

따라서 구현 착수는 아래 조건이 닫힌 뒤에만 진행한다.

```text
시계버스 graybox
→ 첫 2분 재미 게이트
→ 접근성 대체 조작
→ 360px 화면 검증
→ 그 뒤 route와 저장 확장
```

## 2. 구현 티켓

| ID | 티켓 | 목표 |
| --- | --- | --- |
| R-01 | 기존 시계탑 개선 중단 | 카드형 문제풀이 방향으로 더 고치지 않음 |
| R-02 | 수리 스테이지 상태 모델 | 장치, 부품, 슬롯, 작동, 반응 상태를 타입으로 고정 |
| R-03 | 시계버스 graybox | 아트 polish 없이 첫 장치를 만질 수 있게 구현 |
| R-04 | 부품 선택 대체 조작 | 끌기 없이 `부품 선택 -> 슬롯 선택 -> 작동` 가능 |
| R-05 | 장치 반응 엔진 | 부족, 과다, 오배치, 성공 반응을 문구보다 먼저 표시 |
| R-06 | 시간 렌즈 토글 | 보상 카드가 아니라 장치 구조를 드러내는 입력 방식 |
| R-07 | 전이 장치 | 같은 시간 차이를 다른 시간표에 1회 적용 |
| R-08 | 브라우저 저장 | 완료 id, 능력 id, 단서 id, 설정만 저장 |
| R-09 | 접근성 경로 | 키보드, 터치, 큰 글자, 모션 축소 완료 |
| R-10 | 재미 게이트 | 첫 2분 수동 검증 기준 통과 |
| R-11 | route 복구 | `/codaro/math-city` 직접 진입과 새로고침 확인 |
| R-12 | 확장 보류 해제 | 시계버스 통과 후에만 지도, 도감, 단서장 polish |

## 3. 수리 스테이지 인수조건

### R-02 상태 모델

- 공개 상태는 `idle`, `partSelected`, `partPlaced`, `readyToRun`, `running`, `recoverableWrong`, `success`, `repaired`를 가진다.
- 장치 오브젝트는 상태를 가진다.
- 판정은 화면 컴포넌트가 아니라 수리 규칙 함수에서 수행한다.
- 오답은 `wrongIdea`와 연결된다.

### R-03 시계버스 graybox

- 첫 화면에서 2:40 시계, 3:00 출발판, 10분 부품이 보인다.
- 아이가 첫 30초 안에 부품을 선택할 수 있다.
- `작동` 버튼을 누르면 장치가 움직인다.
- 부품 1개, 2개, 3개에 대해 서로 다른 반응이 있다.

### R-04 대체 조작

- 끌기 없이 완료 가능하다.
- 부품 버튼은 48px 이상이다.
- 슬롯 버튼은 이름과 상태를 가진다.
- 키보드 포커스 순서는 장치 설명, 부품, 슬롯, 작동 순서다.

### R-05 장치 반응

- 부족은 목표 직전에서 멈춘다.
- 과다는 목표를 지나쳤다가 되돌아온다.
- 성공은 장치가 자연스럽게 작동한다.
- 문구는 반응 뒤에 45자 안팎으로 나온다.

### R-06 시간 렌즈

- 시간 렌즈는 완료 후 실제 토글로 설치된다.
- 토글을 켜면 두 시각 사이 슬롯이 빛난다.
- 도감 카드는 보조 설명일 뿐 핵심 보상이 아니다.

### R-10 재미 게이트

- 첫 2분 안에 실패와 성공을 모두 경험한다.
- 읽기 시간보다 조작 시간이 길다.
- 수동 관찰에서 “하나 더 넣어야 하나?” 같은 예측 반응이 나온다.
- “또 문제야?” 반응이 먼저 나오면 실패다.

## 4. Gate 목록

| Gate | 실패 조건 |
| --- | --- |
| `math-city-repair-contract` | 수리 스테이지 필수 필드 누락, 장치 상태 없음 |
| `math-city-repair-browser` | 첫 조작, 작동, 실패, 성공 경로 실패 |
| `math-city-repair-accessibility` | 키보드 완료 불가, 대체 조작 없음, 큰 글자 overflow |
| `math-city-repair-visual` | 360/390/414/768/1280 폭에서 장치와 조작대가 겹침 |
| `math-city-storage` | 저장 실패 시 플레이 중단, 개인정보성 필드 저장 |
| `math-city-fun-gate` | 첫 2분 수동 검증 실패 |

## 5. 구현 보류선

아래는 시계버스 graybox가 통과하기 전에는 하지 않는다.

- 에피소드 2 추가.
- 도감 polish.
- 단서장 polish.
- 스토리북 export 구현.
- service worker.
- 소리와 배경 음악.
- 자유 이동 지도.

## 6. 완료 판정

리부트 구현 1차 완료는 다음을 의미한다.

```text
시계버스 수리 장치 하나가
첫 2분 재미 게이트,
접근성 경로,
모바일 화면,
브라우저 저장을 통과했다.
```

그 전까지는 제품 구현 완료가 아니라 graybox 검증 단계다.
