# 시각 및 상호작용 방향

> 리부트 메모: 시각 목표는 유지하되, 화면 중심은 지도/패널이 아니라 라이브 수리 스테이지다. 장면 속 장치, 부품, 슬롯, 작동 버튼이 첫 위계다.

## 1. 시각 목표

수상한 수학도시는 학습지를 예쁘게 꾸민 화면이 아니라, 아이가 직접 고치는 작은 도시여야 한다.

프리미엄 제작 단계와 visual QA는 [premiumProductionPipeline.md](premiumProductionPipeline.md)를 따른다.

핵심 화면 감정:

```text
이상하다
단서가 보인다
내가 조작해본다
도시가 달라진다
다음 장소가 궁금하다
```

## 2. 첫 화면

첫 화면은 랜딩 페이지가 아니다. 바로 도시 지도다.

첫 화면 구성:

- 상단: 게임 제목, 설정 버튼, 진행 초기화 버튼.
- 중앙: 도시 지도.
- 지도 위 장소: 시계탑, 버스 정류장, 빵집, 잠긴 다리, 잠긴 공방.
- 하단: 현재 열 수 있는 에피소드 카드 1개.
- 우측 또는 하단 탭: 능력 도감, 단서장.

첫 화면에서 보여야 하는 것:

- 아이가 어디를 눌러야 하는지.
- 어떤 장소가 고쳐졌는지.
- 다음 장소가 왜 잠겨 있는지.

## 3. 지도 상태

지도는 진행률 숫자보다 강하다. 장소는 상태를 가진다.

| 상태 | 시각 표현 |
| --- | --- |
| locked | 흐린 색, 작은 자물쇠, 장소 실루엣 |
| available | 부드러운 빛, 흔들리지 않는 작은 강조 |
| corrupted | 어긋난 선, 빠진 숫자, 멈춘 오브젝트 |
| restored | 색이 돌아오고 움직임이 생김 |

Core Slice 지도 장소:

- 시계탑 광장
- 숫자 버스 정류장
- 반쪽 빵집
- 길이 다리
- 패턴숲
- 도형 공방

지도는 한 화면에 과하게 많은 장식을 넣지 않는다. 초등학생이 장소를 바로 구분할 수 있어야 한다.

## 4. 장면 화면

에피소드 화면은 다음 구조를 가진다.

```text
[장면 그림]
[짧은 대화]
[단서 또는 문제 카드]
[선택지 또는 조작 영역]
[힌트 버튼]
```

카드만 쌓는 방식은 피한다. 배경 장면 안의 오브젝트가 실제 단서여야 한다.

예:

- 시계탑 장면에서 시계 바늘과 버스 시간표가 동시에 보인다.
- 버스 장면에서 번호판의 빈칸이 실제로 비어 있다.
- 빵집 장면에서 손님 4명과 케이크 1개가 한눈에 보인다.

## 5. 개념 능력 표현

능력은 작은 도구 아이콘과 화면 효과를 가진다.

| 능력 | 아이콘 방향 | 화면 효과 |
| --- | --- | --- |
| 시간 렌즈 | 둥근 렌즈와 시계 눈금 | 두 시각 사이가 색으로 이어짐 |
| 패턴 렌즈 | 반복 화살표와 작은 렌즈 | 규칙 후보가 순서대로 빛남 |
| 나누기 칼 | 둥근 케이크와 안전한 칼 | 전체 위에 같은 간격 가이드가 생김 |
| 합치기 장갑 | 두 조각을 모으는 장갑 | 흩어진 조각이 붙음 |
| 거리 줄자 | 접힌 줄자 | 길이 칸이 측정됨 |

도감에서는 개념명도 함께 보여준다.

예:

```text
시간 렌즈
개념: 시각과 시간 차이
쓸 수 있는 곳: 시계탑, 버스 시간표
```

## 6. 성공 애니메이션

성공은 큰 폭죽보다 세계 변화가 중심이다.

예:

- 시계탑: 바늘이 목표 시각으로 부드럽게 움직이고 종이 한 번 울림.
- 버스: 번호판 숫자가 돌아오고 노선 선이 이어짐.
- 빵집: 조각들이 같은 크기로 맞춰지고 손님들이 웃음.

애니메이션은 1-2초 안에 끝난다. 다음 행동을 막는 긴 연출은 피한다.

## 7. 오답 시각 피드백

오답은 흔들림, 색 차이, 되돌아감으로 표현한다.

원칙:

- 빨간색만으로 실패를 표시하지 않는다.
- 아이를 꾸짖는 문구를 쓰지 않는다.
- 오답 이유가 시각적으로 보여야 한다.

예:

- 공평하지 않은 케이크 조각은 크기 차이가 선으로 표시된다.
- 잘못된 규칙의 버스 번호는 앞뒤 번호와 간격이 다르게 보인다.
- 시간 차이를 틀리면 시계 바늘이 목표보다 짧거나 길게 움직인다.

## 8. 색과 분위기

색상은 한 가지 색 계열에 갇히지 않는다.

권장:

- 배경: 밝은 회색 또는 부드러운 밤색 계열을 장면별로 변화.
- 시계탑: 청록, 노랑, 회색.
- 버스 정류장: 초록, 빨강, 크림색.
- 빵집: 노랑, 체리색, 밝은 나무색.
- 복구 효과: 따뜻한 빛.
- 이상현상: 보라색만 쓰지 않고, 어긋난 선과 빠진 형태로 표현.

금지:

- 전체 화면이 보라색 또는 남색 계열로만 보이는 것.
- 장식용 흐림 원형 배경.
- 텍스트가 이미지 위에서 읽히지 않는 것.
- 카드 안에 또 카드를 넣는 구조.

## 9. 모바일 기준

모바일을 1차 기준으로 본다.

기준:

- 360px 폭에서 모든 버튼 텍스트가 잘리지 않는다.
- 주요 선택지는 엄지로 누르기 쉬운 높이를 가진다.
- 지도는 확대 없이 장소 3-6개가 구분되어야 한다.
- 장면과 문제 영역은 세로 스크롤 하나로 이어진다.
- 도감과 단서장은 하단 탭 또는 시트로 연다.

데스크톱에서는 지도와 도감, 단서장을 옆으로 배치할 수 있다.

## 10. 접근성

- 모든 문제는 색만으로 답을 구분하지 않는다.
- 주요 버튼은 키보드로 접근 가능해야 한다.
- 모션 줄이기 설정을 제공한다.
- 글자 크게 보기 설정을 제공한다.
- 소리는 기본 꺼짐으로 시작해도 된다.
- 이미지 단서에는 화면 텍스트로 같은 정보가 제공되어야 한다.

## 11. 에셋 전략

Core Slice는 과한 일러스트 제작보다 명확한 형태가 중요하다.

권장 순서:

1. CSS와 간단한 SVG 형태로 지도와 오브젝트를 만든다.
2. 필요한 핵심 장면만 별도 bitmap 일러스트로 만든다.
3. 반응형과 상태 변화가 안정화된 뒤 장면별 디테일을 늘린다.

게임 에셋은 재사용 가능한 파츠로 둔다.

예:

- 장소 아이콘
- 능력 아이콘
- 캐릭터 표정
- 단서 쪽지
- 복구 빛 효과

## 12. 프리미엄 금지선

- 지도 전체를 통 bitmap으로 만들지 않는다.
- 핵심 수학 오브젝트를 상태 없는 이미지로 만들지 않는다.
- 장면보다 카드 UI가 먼저 보이게 하지 않는다.
- 하단 조작판이 핵심 장면을 완전히 가리지 않는다.
- `graybox`가 접근성과 모바일 검증을 통과하기 전에는 아트 polish로 넘어가지 않는다.
- visual QA 없이 “보기 좋다”는 주관 판단만으로 출시하지 않는다.
