# 03. Remote Access P2P - WebRTC 개인 클라우드와 보안 모델

상태: 상세 설계 v0.1 (2026-07-10)
범위: 외부 네트워크(폰·회사 브라우저)에서 내 로컬 엔진에 무료로 붙는 P2P 경로, 페어링·권한·차단 보안 모델, 검증 항목.

---

## 1. 목표와 비목표

- 목표: "네 클라우드는 네 컴퓨터다." 집 PC의 로컬 엔진을, 서버 비용 0으로, 외부 브라우저에서 로컬급 그대로 쓴다. 모바일 대응(현재 우선순위 5번)의 실행 기반이기도 하다.
- 비목표: 다인 공유 호스팅(타인에게 내 엔진을 여는 것), 상시 공개 URL. 이건 개인 1인의 원격 접속이다.

## 2. 아키텍처

```text
외부 브라우저(웹 표면)                          집 PC (런처+엔진 상주)
  RTCPeerConnection  <== DataChannel P2P ==>  RTCPeerConnection
        |                                         |
        +---- 시그널링(offer/answer 교환, 얇게) ---+
              STUN: 무료 공용 (NAT 홀펀칭)
              TURN: 대칭 NAT 폴백 릴레이 (Phase 0 census: 무료 한도)
```

- 데이터 채널 위에 기존 API 의미론을 태운다: 엔진의 HTTP/WS 표면을 채널 멀티플렉싱으로 프록시(로컬 브리지에서 검증한 것과 같은 계약, 전송만 교체). 라우터 입장에서는 "local 티어 연결됨"과 동일하게 보인다.
- 시그널링은 상태 없는 offer/answer 교환만 필요하다. 1차 후보 = Cloudflare Workers 무료(카드 불필요, 일 10만 요청)로 초경량 교환기. 완전 무설정 폴백 = 방 코드 수동 입력(QR로 offer 전달). 서버에 실행 데이터가 지나가지 않는 구조를 유지한다(시그널링에는 SDP만).

## 3. 보안 모델 (이 문서의 본체)

위협 정의: 이 기능은 본질적으로 "내 PC에서 임의 파이썬을 실행할 권한"을 네트워크로 여는 것이다. 방 코드 탈취 = 원격 코드 실행. 따라서 기본값은 전부 닫힘이다.

### 3.1 페어링
- 단기 방 코드(QR): 짧은 TTL(분 단위)·1회용. 코드 입력만으로는 연결 완료가 아니다.
- 승인 게이트: 새 기기 최초 연결 시 집 PC 트레이에 승인 알림(기기명·지문 표시). 승인 전 어떤 API도 열리지 않는다.
- 장기 신뢰: 승인 시 기기별 키 쌍을 교환·저장(재방문 시 코드 불필요). 기기 목록·해지는 트레이/설정 표면에서.

### 3.2 권한 스코프 (기본 최소)
- v1 기본 허용: 커널 실행·문서 읽기/쓰기(워크스페이스 안)·커리큘럼 API.
- 명시 옵트인으로만 확장: 터미널(PTY), 자동화 조작(런/E-Stop), 파일시스템 전역 API. 각 스코프는 기기별로 기억된다.
- 워크스페이스 경로 격리(resolvePath, src/codaro/system/fileOps.py:52-71)는 원격 세션에도 동일 적용.

### 3.3 차단과 가시성
- E-Stop 연동: 기존 자동화 비상 정지 표면에서 "원격 세션 전체 끊기"를 1급 동작으로.
- 원격 세션은 감사 로그(기존 자동화 audit JSONL 패턴)를 남긴다: 언제·어느 기기가·어떤 스코프를 썼는지.
- 유휴 TTL: 무활동 세션 자동 만료. 재연결은 저장된 기기 키로 무마찰.

### 3.4 명시적 비설계
- 비밀번호 인증(재사용·유출 위험, 기기 키가 대체), 공개 접속 URL, 중계 서버에 엔진 API를 노출하는 relay-as-a-service. TURN 릴레이는 암호화된 DTLS 패킷만 지나간다(내용 접근 불가).

## 4. Phase 0 census (착수 전 검증 항목)

1. TURN 무료 경로: Cloudflare 무료 TURN 티어의 현행 한도·카드 요구 여부 실측. 불충족 시 폴백 = "동일 네트워크(집 안 Wi-Fi) 한정 v1"로 축소하고 TURN은 후속.
2. 시그널링 워커의 무료 지속 조건(카드 불필요 확인 포함).
3. 모바일 브라우저(Android Chrome 우선)에서 DataChannel 처리량·에디터 표면 실사용성.
4. 기존 연결 상태 스토어(connectionStatus, 재연결 바)와의 통합 지점 확인 - 원격 끊김도 같은 UX 어휘로.

## 5. 착지 기준

- 성공: 같은 계정의 폰 브라우저에서 방 코드 페어링 후, browserRunnable 여부와 무관하게 전 셀이 집 PC에서 실행되고 결과가 스트림된다. 승인 없는 기기는 어떤 경로로도 실행 불가.
- 롤백: 이 기능은 옵트인 모듈이다. 시그널링 워커 제거 + 트레이 토글 오프 = 기능 전체 비활성, 로컬/브라우저 티어는 무영향([06 §4](06-scope-phasing-guardrails.md)).
