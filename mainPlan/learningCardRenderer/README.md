# 학습카드 렌더러 개선 기획 인덱스

이 문서 묶음은 Codaro 본체의 전체 UI 개편이 아니라, 커리큘럼 학습카드 렌더러만 다루는 별도 기획이다. 문제의 중심은 카드 종류 부족이 아니라 소개 카드와 섹션 카드가 학습자의 시선을 한 흐름으로 이끌지 못한다는 점이다.

## 핵심 결정

- 범위: 현재 학습 표면의 소개 카드, 구조화 섹션 카드, 보조 블록 카드, 실행 결과, 검증 피드백.
- 제외: 사이드바 전체 개편, 노트북 표면 개편, 커리큘럼 YAML 대량 수정, 새 카드 type 대량 추가.
- 원칙: `learningContract`와 `sectionContract`를 1차 입력으로 삼고, legacy `sections[].blocks[]`는 보조 자료 렌더러로 남긴다.
- 시각 톤: zinc/shadcn token을 기본으로 하고, 색은 상태와 위험도 표식에만 쓴다.
- 배치 기준: 한 화면에는 하나의 primary action만 두고, grid는 긴 한국어 문장을 강제로 균등 카드에 밀어 넣지 않는다.
- 북극성: 학습자가 섹션 카드 하나에서 무엇을 읽고, 어디를 고치고, 어떤 결과로 확인하는지 10초 안에 이해한다.
- 품질 기준: 데스크톱과 모바일에서 긴 한국어 제목, 긴 설명, 코드, 검증 피드백이 겹치거나 잘리지 않는다.
