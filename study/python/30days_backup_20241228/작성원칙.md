## ⚠️ 최우선 핵심 원칙 (절대 준수)

### 🔴 1. 섹션 간 변수 재할당 절대 금지
**marimo 환경에서 학생들이 서로 다른 셀에서 각 섹션을 실행하기 때문에, 변수명이 중복되면 오류가 발생합니다.**

```python
# ❌ 잘못된 예 - 여러 섹션에서 같은 변수명 사용
## 섹션1: 리스트 생성
fruits = ['apple', 'banana']

## 섹션2: 리스트 추가
fruits = ['apple', 'banana']  # ❌ 재할당! 학생이 섹션2만 실행하면 에러
fruits.append('orange')

# ✅ 올바른 예 - 각 섹션마다 고유한 변수명
## 섹션1: 리스트 생성
createFruits = ['apple', 'banana']

## 섹션2: 리스트 추가
appendFruits = ['apple', 'banana']
appendFruits.append('orange')
```

**변수명 패턴**: 섹션 주제 + 타입명 (예: `createFruits`, `indexFruits`, `sliceFruits`, `modifyFruits`)

### 🔴 2. text 설명 반드시 강화
- **처음 나오는 전문 용어는 반드시 상세하게 설명**
- 용어의 뜻, 유래, 실생활 비유 포함
- 왜 필요한지 동기 부여
- 최소 3~5문장 이상으로 충분히 설명

### 🔴 3. text → list → code → tip 순서 엄수
**필수**: text, list, code는 **모든 섹션에 반드시 포함**
**선택**: tip은 필요할 때만 사용

### 🔴 4. 개념 순서 절대 준수
- **파일 내에서 아직 배우지 않은 개념을 미리 사용하지 않음**
- **이전 day에서 배운 내용은 사용 가능**
- **나중 day에서 배울 내용은 절대 사용 금지**
- 같은 파일 내에서도 뒤 섹션에서 배우는 것을 앞 섹션에서 사용 금지

예시:
```python
# day02_변수.yaml에서
# ✅ 가능: print(), type() (이미 배운 내장함수)
# ❌ 금지: list, dict (day05, day08에서 배움)

# day05_리스트.yaml 섹션3에서
# ❌ 금지: append() (섹션6에서 배움)
# ✅ 가능: 인덱싱 (섹션2에서 배움)
```

### 🔴 5. expansion 복습 원칙
- **당일 배운 모든 내용 + 이전 day 배운 내용 모두 포함**
- 그날 전체 학습 내용을 종합적으로 복습할 수 있어야 함
- 미션 형태로 구성 (미션1, 미션2, ...)

### 🔴 6. 전문 용어 설명 의무
**처음 등장하는 전문 용어는 반드시 text 블록에서 상세 설명**
- 용어의 정의
- 영어 뜻과 유래
- 왜 그렇게 불리는지
- 실생활 비유

### 🔴 7. 모듈 이후 import 관리 원칙
**모듈(day13) 이후부터 import를 사용하는 경우:**
- **파일 최상단에 import 섹션을 하나만 생성**
- **그날 학습에 필요한 모든 import를 한 곳에 모음**
- **이후 섹션에서는 절대 import 사용 금지**

```python
# ✅ 올바른 예 - 최상단 import 섹션
## import 섹션
import math
import random
from datetime import datetime

## 섹션1: math 사용
result = math.sqrt(16)

## 섹션2: random 사용
num = random.randint(1, 10)

# ❌ 잘못된 예 - 각 섹션마다 import
## 섹션1
import math
result = math.sqrt(16)

## 섹션2
import random  # ❌ 중복 import!
num = random.randint(1, 10)
```

## 패턴학습 파일 작성 원칙

패턴학습 파일(day02_2, day05_2 등)은 정규학습 파일의 복습용으로, 간결하게 작성합니다.

### 패턴학습 핵심 규칙
1. **code 블록 필수** - 핵심 패턴 코드는 반드시 포함
2. **text 블록 추가 가능** - "해당 개념을 복습합니다" 정도의 간단한 설명 추가 가능
3. **list 블록 생략** - 패턴만 보여주면 되므로 list는 불필요
4. **tip 블록 추가 가능** - 유용한 팁이 있으면 추가
5. **변수 재할당 금지** - 정규학습과 동일하게 섹션별 고유 변수명 사용
6. **import 관리** - 모듈 이후에는 최상단 import 섹션만 사용

### 패턴학습 구조
```yaml
- id: section_name
  title: 패턴 제목
  subtitle: 간단한 설명
  blocks:
  - type: text  # 선택 - 간단한 설명 추가 가능
    content: 이 개념을 복습합니다.
  - type: code  # 필수
    language: python
    title: 제목
    description: 이 패턴을 복습합니다.
    content: |-
      uniqueVar = "example"
  - type: tip  # 선택
    content: 유용한 팁
```

## 섹션 구조 (필수)

하나의 섹션에는 다음 순서로 블록이 포함되어야 한다:

1. **text** - 개념 설명 (필수, 3~5문장 이상)
2. **list** - text 내용 요약 (필수, 3~5개 항목)
3. **code** - 예제 코드 1개 (필수)
4. **tip** - 꿀팁 (선택)
